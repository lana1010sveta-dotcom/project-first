# Квиз-лендинг «Ищем недвижимость в РБ?» — спецификация

*Создана по итогам discovery-интервью 19.05.2026*

---

## Executive Summary

Квиз-лендинг с разветвлённой логикой для лидогенерации риелтора, специализирующегося на рынке недвижимости Беларуси. Пользователь отвечает на 5–8 вопросов в зависимости от цели, оставляет контакт — риелтор получает структурированный профиль клиента в Telegram. Инфраструктура бесплатная: Netlify + Netlify Functions + Telegram Bot API + Resend.

---

## Статус согласования

**Риелтор не видел ни паспорт, ни вопросы квиза.** До начала вёрстки нужно:
1. Показать риелтору структуру квиза и тексты вопросов
2. Получить цвета/фото или одобрить предложенный дизайн
3. Уточнить имя/название (сейчас плейсхолдер)
4. Договориться по домену

После шага 1 («Что вы хотите сделать?») квиз разветвляется на 3 ветки:

### Ветка A — Купить / Арендовать (7–8 шагов)

| Шаг | Вопрос | Варианты |
|-----|--------|----------|
| 1 | Что вы хотите сделать? | Купить / Продать / Арендовать / Проконсультироваться |
| 2 | Тип объекта? | Квартира / Дом / Участок / Коммерческая |
| 3 | Локация? | Минск / Минская область / Другие области / Не важно |
| 4 (Купить) | Бюджет? | До $50К / $50–100К / $100–200К / Выше / Обсудим |
| 4 (Арендовать) | Бюджет/мес? | До $300 / $300–600 / Выше / Обсудим |
| 5 | Сроки? | Срочно (до 1 мес) / До 3 месяцев / Присматриваюсь |
| 6 | Текущая ситуация? | Только начинаю / Уже смотрю варианты / Есть одобренная ипотека |
| 7 | Что важнее всего? | Цена / Скорость сделки / Юридическая чистота / Ремонт и инфраструктура |
| 8 | Комментарий (опционально) | Свободный текст + кнопка «Пропустить» |
| → | Лид-гейт | Имя / Телефон / Email / Удобное время |

| Шаг | Вопрос | Варианты |

| → | Лид-гейт | Имя / Телефон / Email / Удобное время |

| Шаг | Вопрос | Варианты |
|-----|--------|----------|

| → | Лид-гейт | Имя / Телефон / Email / Удобное время |

## Лид-гейт (финальная форма)

Поля:
- **Имя** — текст, обязательное
- **Телефон** — обязательное, выбор страны +375 / +7, маска номера
- **Email** — необязательное
- **Удобное время** — радиокнопки: Утро / День / Вечер

Кнопка: **«Отправить»** → блокируется после нажатия (защита от дублей)

Чекбокс согласия: «Соглашаюсь на [обработку персональных данных](/privacy)» — обязательный

## Хранение прогресса (localStorage)

Сохраняем при каждом шаге:
```js
{
  goal: "buy",        // ветка: buy / sell / rent / consult
  step: 3,            // текущий шаг
  answers: { ... }    // объект с ответами
}
```

При возврате на страницу — предлагаем продолжить или начать заново. При отправке формы — очищаем хранилище.

## Валидация телефона

- Дропдаун: 🇧🇾 +375 / 🇷🇺 +7
- Маска BY: +375 (XX) XXX-XX-XX (9 цифр)
- Маска RU: +7 (XXX) XXX-XX-XX (10 цифр)
- Форма не отправляется без корректного номера

## Netlify Function

**POST /api/submit**

Принимает JSON с ответами и контактами.
Делает два параллельных запроса:
1. Telegram Bot API — отправляет структурированное сообщение
2. Resend API — дублирует на email риелтора

Обработка ошибок:
- Если оба упали → показать «Что-то пошло не так, попробуйте ещё раз»
- Если упал только один → всё равно показать страницу «Спасибо» (данные дошли хотя бы по одному каналу)

## Юридика

- Страница `/privacy` — политика конфиденциальности (закон РБ №99-З)
- Чекбокс согласия на форме — обязательный, отправка заблокирована без галочки
- В политике упомянуть: данные уходят в Telegram (третья сторона)
- Email-маркетинг: если планируется — отдельный чекбокс (не в MVP)

---

## Out of Scope (MVP)

- Google Sheets — второй этап
- CRM-интеграция — при объёме 50+ заявок/мес
- A/B-тестирование — после первых данных
- Мультиязычность (EN/BY)
- Калькулятор ипотеки
- Личный кабинет
- Условная логика внутри веток (в MVP — линейная последовательность)
