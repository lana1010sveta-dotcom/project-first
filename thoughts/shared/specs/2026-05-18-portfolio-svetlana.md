# Сайт-портфолио веб-разработчика — Спецификация

## Краткое описание

Сайт-визитка Светланы — веб-разработчика, создающего визитки, лендинги, чат-ботов, квизы и полноценные сайты. Цель сайта — привлечь заказчиков через поиск и соцсети, показать портфолио и мотивировать немедленно оставить заявку.

---

## Проблема

Светлана предоставляет услуги по созданию цифровых продуктов, но у неё нет единого места, куда можно отправить потенциального клиента. Сайт должен закрывать эту задачу: демонстрировать экспертизу, вызывать доверие и конвертировать посетителя в заявку.

---

## Метрики успеха

- Посетитель оставляет заявку через форму или переходит в мессенджер
- Сайт находится в поиске по профессиональным запросам
- Новые работы добавляются в портфолио без помощи стороннего разработчика

---

## Целевая аудитория

**B2C** — частные лица, которым нужна визитка, лендинг или квиз  
**B2B** — небольшие бизнесы и предприниматели, ищущие подрядчика для сайта или чат-бота  
**Язык:** русский (с заделом на расширение)  
**Канал прихода:** Google/Яндекс + ссылки из Telegram/Instagram

---

## Путь пользователя

1. Посетитель открывает сайт из поиска или по ссылке
2. Видит Hero-секцию: имя, специализация, яркий CTA «Обсудить проект»
3. Листает вниз: портфолио работ, услуги с ценами, отзывы клиентов
4. Нажимает кнопку связи **или** заполняет форму
5. Получает ответ от Светланы

## Структура сайта

### Одностраничный сайт (Landing Page) + 2 юридические страницы

| Секция | Содержимое |
|---|---|
| **Hero** | Имя, должность, короткий оффер, кнопки «Написать в Telegram» и «WhatsApp» |
| **О себе** | Фото, краткая биография, ключевые компетенции, опыт |
| **Портфолио** | Карточки работ с фильтрацией по категориям |
| **Услуги и цены** | Перечень услуг с описанием и ценой или диапазоном |
| **Отзывы** | Карточки с именем клиента, текстом, опционально — аватар |
| **Контакты** | Форма заявки + иконки мессенджеров + телефон/email |
| **Footer** | Реквизиты, ссылки на юр. страницы, копирайт |

### Юридические страницы
- `/privacy` — Политика конфиденциальности (ФЗ-152)
- `/terms` — Пользовательское соглашение / Условия оказания услуг

## Технические требования

### Стек

| Компонент | Решение | Причина |
|---|---|---|
| Фреймворк | **Vite + React** или **чистый HTML/CSS/JS** | Лёгкость сборки, быстрая загрузка |
| Стили | **CSS Modules** или Tailwind CSS | Изолированные стили, удобно при вайбкодинге |
| Данные портфолио | `src/data/portfolio.json` | Легко редактировать через Claude Code |
| Данные услуг | `src/data/services.json` | Аналогично |
| Данные отзывов | `src/data/reviews.json` | Аналогично |
| Хостинг | **Netlify** (бесплатно) | Автодеплой из GitHub, CDN, HTTPS |
| Форма | Netlify Forms или Formspree | Без бэкенда, письмо приходит на email |

### Структура JSON для портфолио

```json
[
  {
    "id": 1,
    "title": "Визитка для фотографа",
    "category": "vizitka",
    "description": "Одностраничный сайт-визитка с формой записи",
    "image": "/portfolio/foto-vizitka.jpg",
    "url": "https://example.com",
    "tags": ["HTML", "CSS"]
  }
]
```

Категории: `vizitka` | `lending` | `chatbot` | `quiz` | `site`

### Обновление портфолио

Добавить новую работу = отредактировать `portfolio.json` через Claude Code:  
_«Добавь в портфолио новый проект: название X, категория Y, ссылка Z, описание...»_

## Дизайн

### Стиль: Градиент + Живой

- **Цветовая схема:** яркие градиенты (фиолетово-синий, малиново-оранжевый или подобное) на тёмном/светлом фоне
- **Типографика:** современный sans-serif (Inter, Nunito или Manrope)
- **Анимации:** плавное появление секций, hover-эффекты на карточках, micro-interactions на кнопках
- **Карточки портфолио:** скриншот + оверлей с кнопкой «Посмотреть» при наведении
- **Адаптивность:** mobile-first, breakpoints: 375px / 768px / 1200px

## Юридические требования

### Политика конфиденциальности (`/privacy`)
- Описание, какие данные собираются (имя, телефон/email через форму)
- Цель сбора — связь с пользователем
- Ссылка на ФЗ-152 «О персональных данных»
- Контактные данные оператора данных

### Пользовательское соглашение (`/terms`)
- Описание услуг
- Порядок оказания услуг и оплаты
- Ограничение ответственности

### В форме обратной связи
- Обязательный чекбокс: «Я согласен(а) на обработку персональных данных» со ссылкой на политику
- Форма не отправляется без чекбокса

### Footer
```
ИП Николаева Светлана [Отчество] 
ИНН: [заполнить после регистрации]
ОГРНИП: [заполнить после регистрации]
```

> **Примечание:** До регистрации как ИП реквизиты можно временно не показывать или указать «Самозанятая».

## Открытые вопросы для реализации

1. **Форма → куда?** Netlify Forms (проще) vs Telegram-бот (удобнее получать заявки). Рекомендую стартовать с Netlify Forms.
2. **Домен** — есть ли уже имя домена? Если нет — выбрать перед деплоем.
3. **Фото** для секции «О себе» — нужно подготовить профессиональное фото.
4. **Реквизиты** — заполнить ИНН/ОГРНИП после регистрации ИП.
5. **Тексты** — написать оффер (Hero), биографию, описания услуг с ценами, тексты юридических страниц.

## Сценарий запуска

```
1. Написать тексты всех секций (офлайн)
2. Подготовить скриншоты 3–5 работ для портфолио
3. Открыть Claude Code → «Реализуй спецификацию portfolio-svetlana»
4. Задеплоить на Netlify (бесплатно, из GitHub)
5. Подключить домен
6. Добавить реквизиты после регистрации ИП
```
