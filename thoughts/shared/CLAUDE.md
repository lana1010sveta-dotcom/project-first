## Хостинг и репозитории

- **Vercel** (основной): `my-website-bay-psi.vercel.app` — деплоится автоматически при пуше в `main`

При пуше в `my-website` Vercel деплоит автоматически. Команда:
```

```

> **Деплой запускать из встроенного терминала VS Code** (Terminal → New Terminal) — там
> работает авторизация GitHub через вход VS Code в аккаунт `lana1010sveta-dotcom`.
> Из автоматического/фонового окружения пуш падает с запросом логина. Если git всё же
> просит логин и не проходит — авторизоваться разово через токен доступа (PAT).
> Для этого репозитория `credential.helper` локально переключён на `store` (в обход
> зависающего Git Credential Manager).

## Портфолио — как добавить проект

Найти блок `<!-- PORTFOLIO -->`, добавить карточку в `<div id="pfg">`:

```html
<div class="pc rv" data-t="site">
  <div class="pp" style="background-image:url('img/имя.jpg');background-size:auto 165%;background-position:center 18%"></div>
  <div class="p-tag"><span class="tag t-site">Сайт</span></div>
  <div class="pov">
    <h3 class="p-title">Название проекта</h3>
    <p class="p-meta">Ниша · Описание</p>
    <p class="p-res">✓ Результат</p>
  </div>
</div>
```

Классы карточек: `pc` (обычная), `pc feat` (2×2), `pc wide` (2×1)  
Категории (`data-t`): `site` `landing` `app` `ai`  
Теги (класс): `t-site` `t-land` `t-app` `t-ai`  
Скриншоты класть в `img/`. Параметры фона `background-size:auto 165%; background-position:center 18%` обрезают браузерный хром сверху и тулбар снизу.

**Рабочая ссылка «открыть демо» (стандарт с 28.06.2026):** в `<p class="p-res">` ставим
ссылку на живой проект — как в карточке «Конструктор резюме»:

```html
<p class="p-res">✓ Живой проект · <a href="https://ССЫЛКА" target="_blank" rel="noopener" style="color:#34d399;font-weight:700">открыть демо →</a></p>
```

**Превью карточки:** либо скриншот в `img/` (фон `.pp`), либо брендовая плитка-градиент
с иконкой, когда чистого скриншота ещё нет (как у «Конструктора резюме»).
