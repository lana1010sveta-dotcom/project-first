## 1. СУТЬ ПРОЕКТА

### Одна фраза
**«Сайт-портфолио, который сам продаёт: посетитель приходит → видит результаты → оставляет заявку»**

### Цель
Привлекать заказчиков на создание сайтов, приложений и AI-агентов через органический поиск и ссылки из соцсетей. Главная метрика успеха — входящие заявки от клиентов.

### Автор / Специалист
**Светлана Николаева** — веб-разработчик, создающий сайты, приложения и AI-агентов под ключ.

---

## 2. ЦЕЛЕВАЯ АУДИТОРИЯ

### Первичная (B2C)
**Владелец малого бизнеса, 30–50 лет**
- Знает, что нужен сайт или автоматизация, но не разбирается в технологиях
- Главный страх: «Возьмут деньги и пропадут» / «Не пойму, что мне делают»
- Решение для него: прозрачный процесс, цены на сайте, реальные примеры похожих проектов
- Ищет через Google/Яндекс или получает ссылку от знакомых

### Вторичная (B2B)
**Руководитель / маркетолог компании, 25–45 лет**
- Ищет подрядчика для конкретной задачи: лендинг, бот, MVP
- Сравнивает несколько вариантов, оценивает портфолио и процесс
- Важны кейсы с результатами и чёткие условия работы

### Психологический портрет любого клиента (из экспертной оценки)
> «Мой главный страх — не цена. Это неизвестность. Хорошее портфолио снимает этот страх конкретикой, примерами, живым человеческим голосом в тексте.»

## 3. ПОЗИЦИОНИРОВАНИЕ

### Главный оффер (рекомендация маркетолога: нишевать результат, не продукт)

```
ЗАГОЛОВОК HERO:
«Строю цифровые системы, которые работают вместо вас»

ПОДЗАГОЛОВОК:
Сайты, приложения и AI-агенты под ключ — от идеи до запуска за 2–6 недель
```

### Почему этот оффер
- AI как главный крючок (хайп-фаза спроса, 2025–2026)
- Сайты и приложения остаются как «инфраструктура под систему»
- «Работает вместо вас» — говорит о результате, не о процессе
- Понятно нетехническому клиенту без жаргона

### Альтернативный вариант заголовка (A/B тест)
```
«Ваш первый AI-агент — готов за 2 недели. Делаю под ключ.»
```

### УТП (3 ключевых преимущества)
```
⚡ Быстро    — Запуск за 2–6 недель, не за полгода
🤖 Умно      — AI-интеграция в каждый продукт по запросу
🛡 Надёжно   — Поэтапная оплата, правки включены, поддержка после
```

## 5. СОДЕРЖАНИЕ КАЖДОЙ СЕКЦИИ

### 00. ШАПКА (Sticky Navigation)

**Поведение:** прилипает к верху при скролле, полупрозрачный фон + blur

**Содержимое:**
```
[Логотип / Имя: «СН» или «Svetlana Dev»]    [Портфолио] [Услуги] [Контакт]    [Написать →]
```

**Правила:**
- Максимум 3 якорных ссылки (не 8)
- Кнопка «Написать» — акцентный цвет, видна сразу
- На мобильном: логотип + кнопка-бургер + кнопка «Написать»

**Текстовые блоки:**
- **H1:** «Строю цифровые системы, которые работают вместо вас»
- **Подзаголовок:** «Сайты, приложения и AI-агенты под ключ — от идеи до запуска за 2–6 недель»

**Кнопки:**
- Primary: «→ Написать в Telegram» (акцентный, градиентный)
- Secondary: «Посмотреть работы ↓» (ghost / outline)

- Анимированный градиентный фон (пурпурно-синий, медленная анимация)

- Заголовок: kinetic typography — слова появляются последовательно (0.08s задержка)
- Градиентный фон: медленная анимация смещения (10–15 сек цикл)
- Кнопки: pulse-эффект на primary кнопке (привлекает внимание)

### 06. КАК МЫ РАБОТАЕМ

**Цель:** Убрать страх неизвестности — клиент видит, что его ждёт

**Заголовок:** «Как проходит работа»
**Подзаголовок:** «Вам нужно около 30 минут в неделю — остальное беру на себя»

**Таймлайн (5 шагов):**

```
①─────────────②─────────────③─────────────④─────────────⑤
│             │             │             │             │
Консультация  Бриф и        Разработка    Запуск        Поддержка
(бесплатно)   концепция     (этапы с      и передача    30 дней
              (ТЗ + смета)  промежуточн.  проекта       включена
                            сдачами)
```

**Описание каждого шага:**

| Шаг | Название | Что происходит | Сколько длится |
|-----|----------|----------------|----------------|
| 1 | Бесплатная консультация | Обсуждаем задачу, предлагаю 2–3 решения | 30–60 мин |
| 2 | Бриф и концепция | Согласовываем ТЗ, сроки, стоимость, структуру | 1–3 дня |
| 3 | Разработка | Работаю поэтапно, показываю промежуточный результат | По договору |
| 4 | Запуск | Тестирование, финальные правки, передача доступов | 1–3 дня |
| 5 | Поддержка | Исправление ошибок, консультации, мелкие правки | 30 дней |

### 07. ОТЗЫВЫ

**Цель:** Социальное доказательство — чужое мнение доверяют больше, чем самопродвижению

**Заголовок:** «Что говорят клиенты»

**Формат карточки отзыва (обязательный минимум):**
```
┌──────────────────────────────────────────────┐
│ [Фото]  Имя Фамилия                 ★★★★★    │
│         Должность, Компания                  │
│                                              │
│ «Цитата отзыва — конкретная, с результатом.  │
│  Не "всё понравилось", а "заявки выросли     │
│  вдвое за первый месяц работы сайта"»        │
└──────────────────────────────────────────────┘
```

**Требования к отзывам:**
- Минимум 2–3 отзыва
- Обязательно: имя + фото (или хотя бы инициалы + сфера деятельности)
- Желательно: конкретный результат в цифрах или фактах
- Опционально: видео-отзыв (+79% доверия vs текстовый)

**Если отзывов пока нет:** добавить раздел «Мои проекты в цифрах» с результатами без имён клиентов

**Слайдер:** авто-прокрутка отключена, управление стрелками или свайпом

### 09. FAQ

**Цель:** Снять последние возражения до того, как клиент уйдёт «подумать»

**Заголовок:** «Часто задаваемые вопросы»

**Формат:** Аккордеон (открывается по клику, одно поле за раз)

**Список вопросов и ответов:**

```
❓ Сколько стоит проект?
→ Стоимость зависит от типа и сложности. Сайт-визитка — от 15 000 ₽,
  лендинг — от 30 000 ₽, AI-агент — от 20 000 ₽. Точную цену скажу
  после бесплатной консультации — это займёт 30 минут.

❓ Сколько времени займёт разработка?
→ Сайт-визитка — 3–7 дней. Лендинг — 1–2 недели. Сложный проект
  (приложение или AI-система) — от 4 до 6 недель. Сроки фиксируем
  в договоре и я их соблюдаю.

❓ Что если мне не понравится результат?
→ Работаю поэтапно: вы видите и принимаете каждый этап до следующего.
  Правки до момента утверждения — без ограничений и без доплат.
  Вы не платите за то, что не приняли.

❓ Смогу ли я сам обновлять сайт после запуска?
→ Да. После передачи проекта провожу инструктаж и оставляю инструкцию.
  Большинство клиентов обновляют тексты и фото самостоятельно.

❓ Нужен ли мне свой хостинг и домен?
→ Если есть — используем ваш. Если нет — помогу выбрать и подключить,
  это включено в стоимость. Вы остаётесь полным владельцем сайта.

❓ Как происходит оплата?
→ Поэтапно: часть при старте, остальное при сдаче каждого этапа.
  Предоплата полностью не прошу — это ваши деньги до тех пор,
  пока вы не получили и не приняли работу.

❓ Вы работаете по договору?
→ Да, всегда. Договор фиксирует объём работ, сроки, стоимость
  и условия. Это защищает и вас, и меня.
```

### Приоритет реализации

| Приоритет | Анимация | Где | Технология |
|-----------|----------|-----|------------|
| 🔴 P0 | Micro-interactions на кнопках | Везде | CSS transition |
| 🔴 P0 | Scroll-reveal карточек | Портфолио, услуги | AOS.js |

| 🟡 P1 | Kinetic Typography заголовка | Hero | CSS @keyframes |
| 🟡 P1 | Счётчики цифр | Секция доверия | JS IntersectionObserver |
| 🟡 P1 | Hover 3D на карточках | Портфолио | CSS transform |
| 🟢 P2 | Параллакс фон | Hero | CSS / JS |
| 🟢 P2 | Progress-bar чтения | Шапка | CSS scroll-timeline |

### Код анимаций

```css
/* 1. Живой градиент Hero */
@keyframes gradient-shift {
  0%   { background-position: 0% 50%; }
  50%  { background-position: 100% 50%; }
  100% { background-position: 0% 50%; }
}
.hero {
  background: linear-gradient(-45deg, #7C3AED, #EC4899, #38BDF8, #0F0F1A);
  background-size: 300% 300%;
  animation: gradient-shift 12s ease infinite;
}

/* 2. Kinetic Typography (Hero заголовок) */
@keyframes word-appear {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}
.hero-title .word {
  display: inline-block;
  opacity: 0;
  animation: word-appear 0.5s ease forwards;
}
/* Каждому слову: animation-delay: 0s, 0.1s, 0.2s... */

/* 3. Счётчики */
/* Реализация через JS — IntersectionObserver + requestAnimationFrame */

/* 4. Scroll-reveal (через AOS.js) */
/* data-aos="fade-up" data-aos-delay="100" */
```

### Правила анимаций (из рекомендаций UX-эксперта)
- ❌ **Нельзя:** автовоспроизведение видео/звука
- ❌ **Нельзя:** анимации, блокирующие чтение контента
- ✅ **Можно:** анимации, дополняющие контент (появление при скролле)
- ✅ **Можно:** micro-interactions (hover, focus, click)
- ⚡ **Всегда:** `prefers-reduced-motion` для пользователей с настройкой «меньше движения»

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```
