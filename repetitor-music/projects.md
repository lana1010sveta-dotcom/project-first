## 1. Что это за проект (в одном абзаце)

ИИ-**репетитор по сольфеджио** для мальчика **Лёни** (10 лет, 4 класс муз. школы/колледжа).
Лёня 3 года занимается музыкой: основной инструмент — **кларнет**, дополнительный —
**фортепиано**. Свободно читает ноты. Репетитор нужен как **дополнение к колледжу**:
в полуигровой форме развивать **музыкальный слух** через вокальные упражнения (распевки)
и попутно давать **понимание умом** — что происходит в мелодии и гармонии. Цель — не
дублировать школу, а закреплять и **увлечь**, чтобы не было скучно.

- **Заказчик:** Светлана (бабушка Лёни). Это её клиентский/семейный проект.
- **Формат — две оси (не путать):** **МОЗГ** = «движок-ядро + агент-дирижёр» (см. §5); **ПЛАТФОРМА** = телефон, путь сайт → гибрид Telegram-бот+веб-апп → Capacitor (чистый мини-апп рискован из-за микрофона в вебвью). Подробно в brief.md §7.
- **Язык/методика:** русский, традиция РФ (фиксированное До = C, ступени I–VII).

## 2. Зачем именно так (ключевая идея)

Инструмент сам по себе НЕ развивает: чистую интонацию голосом, внутренний слух,
чувство гармонии (кларнет одноголосный), ладовое чувство «куда тянет звук». Это и есть
зона репетитора. Кларнет in B♭ (звучит на тон ниже записи) — игровой «крючок» про
транспозицию.

## 3. Что уже сделано ✅

1. **Исследование** — разобрали 7 успешных образцов нейрорепетиторов и слухо-тренажёров
   (Khanmigo, Duolingo/Duolingo Music, Yousician/Vanido, EarMaster, Synthesis Tutor,
   Squirrel AI, Perfect Ear). Из каждого взяли по элементу — см. таблицу в [brief.md](brief.md) §2.
2. **Концепт-бриф** — [brief.md](brief.md), **версия 0.2**. Внутри: персонаж «Маэстро Кларнет»,
   педагогика, структура занятия, учебный план из 7 уровней, функции, тех-концепция, MVP.
3. **Критический разбор идеи** → правки курса в v0.2 (см. §5 ниже — это важно).
4. **Прототип ядра** — [prototype/index.html](prototype/index.html). Самодостаточный HTML,
   работает офлайн, без библиотек. Реализовано:
   - выбор целевой ноты (ступени I–VIII до-мажора) + «другая нота»;
   - проигрывание ноты и **дрон-якорь** тоники (бурдон);
   - захват микрофона, **детектор высоты** (автокорреляция + парабольная интерполяция);
   - «дорожка высоты» на канвасе в реальном времени (зелёная зона = цель);
   - вердикт в центах (🎯 чисто / ⬆ выше / ⬇ ниже) по медиане;
   - **защита от октавных ошибок** (октавой выше/ниже = верная нота);
   - ответ можно дать **голосом ИЛИ игрой** на инструменте (один и тот же микрофон).

## 5. Курс-решения, которые НЕЛЬЗЯ потерять (из разбора, v0.2–v0.3)

Это выводы критического анализа — фундамент проекта:

- 🏛️ **АРХИТЕКТУРА (v0.3): «движок-ядро + агент-дирижёр».** Продукт = детерминированный
  движок + Claude-агент сверху, который **дирижирует** занятием через инструменты (tools):
  `сыграть_ноту`, `включить_дрон`, `начать_слушать`→`получить_результат_высоты`,
  `записать_прогресс`, `вспомнить_прошлое`. Это **«мозг»** (одна ось формата); **платформа** —
  отдельная ось (сайт → гибрид бот+веб → Capacitor). Границы агента: НЕ судит высоту сам,
  НЕ лезет в тугие петли реального времени, держать коротким и безопасным для ребёнка,
  учитывать стоимость LLM на каждый ход. Детали — brief.md §7.
- ⚖️ **Разделение ролей:** аудио-движок (детерминированный) — ЕДИНСТВЕННЫЙ судья
  высоты/интервала/аккорда. **Claude НЕ слышит звук и не оценивает интонацию** — он
  получает готовый вердикт движка и подаёт его по-человечески (вопросы, похвала).
  Иначе — галлюцинации и «врёт про чистоту».
- 🎺 **Инструмент как равноправный ввод.** Лёня уверенно играет — пусть отвечает игрой,
  а не только пением. Снимает страх петь в телефон (риск №1!) и строит мост слышу↔играю↔пою.
- 🎵 **Дрон-якорь тоники** во всех вокальных заданиях — без опоры чисто петь нельзя.
- 🎚️ **Калибровка** микрофона и голосового диапазона до упражнений.
- 🛡️ **Защита от октавных ошибок** детектора.
- 🔁 Режим **«эхо»** (зов-ответ) как низкий порог входа.
- 📚 Совместимость с **методикой колледжа** (не учить «по-своему» — будет каша).
- 👵 **Света в контуре:** сводка о занятиях, совместная похвала — иначе ребёнок забросит.
- 🧪 **Сначала валидируем ядро, потом строим контент** (не наоборот).

**Риск №1 — поведенческий, не технический:** согласится ли 10-летний мальчик петь
в телефон. Поэтому инструмент-ввод и эхо-режим так важны.

## 6. С чего продолжать ▶️ (следующие шаги по приоритету)

1. **[БЛОКЕР] Тест прототипа на телефоне Лёни.** Нужен `https` (телефон не пустит к
   микрофону по http/localhost). Два пути на выбор Светы:
   - быстро — туннель через `c:/projects/cloudflared.exe` (временный https-адрес);
   - надёжно — задеплоить папку `prototype/` на Vercel/Netlify.
   *(Деплой/туннель не запускать без явной просьбы Светы — см. правило ниже.)*
   Проверяем 3 вещи: ловится ли его **голос** без вранья; согласен ли **петь**; понятна ли
   ему «дорожка высоты».
2. **Если ядро работает** → разворачиваем уровни 1–2 (тоника/чистота, ступени лада):
   дрон-упражнения, эхо-режим, геймификация (XP/стрики), затем Claude-диалог поверх движка.
3. **Если ядро врёт/неудобно** → правим захват звука/UX (или делаем упор на инструмент-ввод)
   ДО вложений в контент.
4. Ответить на открытые вопросы из [brief.md](brief.md) §9 (имя персонажа, формат проекта,
   нужна ли сводка для Светы).

## 7. Как запустить и где что лежит 🛠️

```
repetitor-music/
├── projects.md          ← этот файл (читать первым)
├── brief.md             ← полный концепт, v0.3
└── prototype/
    └── index.html       ← прототип ядра (детект высоты)
```

**Тест прототипа на ПК** (микрофону нужен localhost/https, не file://):
```bash
cd c:/projects/repetitor-music/prototype
python -m http.server 8753
# открыть http://localhost:8753/
```

**Тест на телефоне** — нужен https (туннель cloudflared или деплой), см. §6.

**Стек (план):**
- **Мозг:** движок-ядро (Web Audio API — детект высоты + проигрывание + дрон, без аудиофайлов)
  + агент-дирижёр Claude (Anthropic) с инструментами, отдельно от движка.
- **Платформа:** сайт (прототип) → гибрид Telegram-бот + веб-апп в реальном браузере (продукт)
  → Capacitor для App Store/Google Play (при росте). Чистый Telegram-мини-апп не берём —
  микрофон в вебвью ненадёжен.
- TTS браузерный на MVP · лёгкое хранилище прогресса.
