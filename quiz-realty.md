# Квиз «Ищем недвижимость в РБ?» — Проектная документация

## Что это

Квиз-лендинг для риелтора: пользователь отвечает на 7–8 коротких вопросов о своём запросе по недвижимости и оставляет контакт. Риелтор получает уведомление в Telegram с готовым профилем клиента и звонит уже «тёплому» лиду — без холодных обзвонов и пустых разговоров.

Инструмент заменяет стандартную форму «оставьте номер телефона», потому что клиент сам описывает свою ситуацию в формате диалога, а риелтор приходит на звонок подготовленным.

---

## Для кого

**Конечный пользователь (клиент риелтора)**
- Возраст: от 25 лет
- География: Беларусь и за её пределами (диаспора, переезжающие)
- Ситуация: хочет купить или продать недвижимость в Беларуси, но не знает, с чего начать, или не хочет тратить время на долгие переговоры

**Заказчик инструмента (риелтор)**
- Специализируется на рынке недвижимости РБ
- Нуждается в стабильном потоке заявок
- Хочет получать заявки с уже собранной информацией, а не «перезвоните мне»

---

## Какую проблему решает

| Боль | Как квиз её закрывает |
|------|-----------------------|
| Клиенты не оставляют заявки на стандартных формах | Формат теста воспринимается как полезное действие, а не как «отдай номер» |
| Риелтор тратит время на уточняющие вопросы при первом звонке | Данные уже собраны: тип объекта, бюджет, сроки, локация |
| Холодные обзвоны дают низкую конверсию | Клиент сам пришёл и прошёл тест — он уже вовлечён |
| Нет системы учёта входящих обращений | Каждая заявка фиксируется в Telegram с меткой времени |

## Основные функции

### 1. Лендинг-заглушка (точка входа)
- Заголовок: «Ищем недвижимость в РБ?»
- Краткое описание пользы (2–3 строки)
- Единственная кнопка: «Пройти тест за 2 минуты»

### 2. Квиз (7–8 шагов)
Вопросы уточняют запрос клиента. Примерный набор:
1. Цель: купить / продать / арендовать
2. Тип объекта: квартира / дом / участок / коммерческая
3. Локация: Минск / область / конкретный город или район
4. Бюджет: диапазон
5. Сроки: срочно / в течение 3 месяцев / присматриваюсь
6. Текущая ситуация: уже смотрю варианты / только начинаю / есть ипотека
7. Что важнее: цена / скорость / юридическая чистота
8. (опционально) Комментарий в свободной форме

Каждый шаг — один вопрос, кнопки-варианты или короткий ввод текста.
Прогресс-бар показывает, сколько шагов осталось.

### 3. Форма сбора контактов (финальный шаг)
- Имя
- Номер телефона (обязательное поле)
- Удобное время для звонка (утро / день / вечер)
- Кнопка «Отправить заявку»

### 4. Уведомление в Telegram
После отправки формы риелтор получает сообщение в Telegram-канал в структурированном виде:
```
📋 Новая заявка с квиза

👤 Имя: Мария
📞 Телефон: +375 29 000-00-00
🕐 Удобное время: вечер

🏠 Цель: купить квартиру
📍 Локация: Минск, Московский район
💰 Бюджет: до 80 000 $
⏱ Сроки: в течение 3 месяцев
✅ Важно: юридическая чистота

🗓 Дата: 16.05.2026, 14:32
```

### 5. Страница благодарности
После отправки заявки — простой экран: «Спасибо! Риелтор свяжется с вами в [указанное время]».

## Чего НЕ будет на старте

| Функция | Причина откладывания |
|---------|----------------------|
| Личный кабинет клиента | Избыточно для MVP, усложняет разработку |
| CRM-интеграция (Bitrix, amoCRM) | Telegram-уведомления закрывают задачу на старте |
| Оплата или бронирование объектов онлайн | Это не маркетплейс, а инструмент захвата лида |
| Каталог объектов с фильтрами | Задача квиза — собрать заявку, а не показывать листинг |
| Мультиязычность | Аудитория русскоязычная, перевод — в следующем этапе |
| Аналитика и A/B тесты | Добавить после запуска, когда будет трафик |
| Автоответчик / чат-бот | На старте достаточно страницы «Спасибо» |
| Административная панель | Управление через код или отдельный этап |

---

## Стек и интеграции (предварительно)

- **Фронтенд:** HTML / CSS / JavaScript (без фреймворка, в стиле существующего проекта)
- **Отправка данных:** Telegram Bot API через `fetch` на клиенте или минимальный serverless-endpoint
- **Хостинг:** GitHub Pages или любой статический хостинг

---

## Критерий готовности MVP

Квиз считается готовым к запуску, когда:
1. Пользователь может пройти все шаги без ошибок
2. Данные приходят риелтору в Telegram в читаемом виде
3. Страница корректно отображается на мобильном телефоне
4. Форма не отправляется без номера телефона
