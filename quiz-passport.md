# Паспорт проекта: «Ищем недвижимость в РБ?»

---

## 1. Название и суть

**Название:** Квиз-лендинг «Ищем недвижимость в РБ?»

Инструмент лидогенерации для риелтора: пользователь отвечает на 7–8 коротких вопросов о своём запросе по недвижимости, оставляет контакт — риелтор получает готовый профиль клиента прямо в Telegram и звонит уже подготовленным.

Квиз заменяет холодные обзвоны и пустые формы «оставьте номер». Клиент сам приходит, сам описывает ситуацию — риелтор получает тёплый лид без лишних вопросов при первом звонке. По данным исследования, квиз-воронки конвертируют в 17 раз лучше, чем статичные формы (47% против 2.8%).

---

## 2. Для кого

### Конечный пользователь — клиент риелтора
- Возраст: от 25 лет
- География: Беларусь + диаспора и переезжающие из-за рубежа
- Ситуация: хочет купить, продать или арендовать недвижимость в РБ, но не знает, с чего начать, или не хочет тратить время на долгие переговоры
- Устройство: преимущественно смартфон (60%+ мобильного трафика в сегменте)

### Заказчик — риелтор
- Специализируется на рынке недвижимости Беларуси
- Нуждается в стабильном потоке входящих заявок
- Хочет приходить на первый звонок уже зная запрос клиента
- Пользуется Telegram как основным рабочим мессенджером

---

## 3. Какую проблему решает

| Боль | Как квиз закрывает |
|------|--------------------|
| Стандартные формы «оставьте номер» — конверсия 1–3% | Формат теста воспринимается как полезное действие; конверсия 30–50% |
| Риелтор тратит первые 10–15 минут звонка на уточнения | Данные уже собраны: цель, тип объекта, бюджет, сроки, локация |
| Холодные обзвоны дают отказ в 90% случаев | Клиент сам пришёл и прошёл тест — он уже вовлечён и открыт |
| Нет системы учёта входящих заявок | Каждая заявка фиксируется в Telegram с меткой времени и деталями |
| Клиенты из-за рубежа не знают, как начать поиск | Квиз структурирует запрос и снижает порог входа в процесс |

## 4. Функции MVP — делаем первым

### 4.1 Лендинг — точка входа
- Заголовок: «Ищем недвижимость в РБ?»
- 2–3 строки о том, что будет
- Социальное доказательство: «N человек уже нашли своё жильё» (добавить после первых заявок)
- Одна кнопка: «Пройти тест за 2 минуты»
- Полная адаптация под мобильный телефон

### 4.2 Квиз — 7–8 шагов
Один вопрос — один экран. Ответы кнопками (без печати). Прогресс-бар.

| Шаг | Вопрос | Варианты |
|-----|--------|----------|
| 1 | Что вы хотите сделать? | Купить / Продать / Арендовать |Проконсультироваться
| 2 | Тип объекта? | Квартира / Дом / Участок / Коммерческая |
| 3 | Локация? | Минск / Минская область / Другие области / Не важно |
| 4 | Бюджет? | До $50K / $50–100K / $100–200K / Выше / Обсудим |
| 5 | Сроки? | Срочно (до 1 мес) / До 3 месяцев / Присматриваюсь |
| 6 | Текущая ситуация? | Только начинаю / Уже смотрю варианты / Есть одобренная ипотека |
| 7 | Что важнее всего? | Цена / Скорость сделки / Юридическая чистота | Ремонт и инфраструктура
| 8 | Комментарий (опционально) | Свободный текст — пропускается кнопкой |

### 4.3 Лид-гейт — форма контактов (последний шаг)
Размещается ПОСЛЕ всех вопросов — в момент максимальной вовлечённости.
- Имя
- Номер телефона (обязательное — основной канал связи риелтора)
- Email (опционально — для последующих рассылок)
- Удобное время для звонка: утро / день / вечер

### 4.4 Уведомление в Telegram
Риелтор получает сообщение в формате:

```
📋 Новая заявка с квиза

👤 Мария
📞 +375 29 000-00-00
⏰ Удобное время: вечер

🏠 Купить квартиру
📍 Минск
💰 $50 000 – $100 000
📅 В течение 3 месяцев
📌 Уже смотрит варианты
✅ Важно: юридическая чистота

💬 «Ищу 2-комнатную, желательно ближе к метро»

🕐 16.05.2026, 18:43
```

### 4.5 Страница благодарности
После отправки — экран: «Спасибо, [Имя]! Риелтор свяжется с вами [в указанное время]». Никакого перенаправления на внешние сайты.

## 5. Функции на будущее

| Функция | Обоснование из исследования |
|---------|-----------------------------|
| Lead scoring (горячий / тёплый / холодный лид) | Позволяет приоритизировать звонки; реализовано у Marquiz, ScoreApp |
| Условная логика / ветвления | Покупатель и продавец видят разные вопросы; повышает релевантность |
| Аналитика по шагам (Yandex Metrica) | Видно, на каком вопросе уходят; данные для оптимизации |
| A/B-тестирование вариантов | Первый квиз редко оптимален; тест заголовков даёт +20–40% конверсии |
| Email-автоматизация | Follow-up письмо с результатами сохраняет бренд в памяти |
| Интеграция с Google Sheets | Архив всех заявок; риелтор видит историю без входа в систему |
| Калькулятор стоимости / ипотеки | Высококонвертирующий инструмент для привлечения продавцов |
| Страница персонализированных результатов | «Вы — покупатель с чётким запросом» — повышает доверие |
| CRM-интеграция (amoCRM / Bitrix24) | Актуально при объёме 50+ заявок в месяц |
| Мультиязычность (EN/BY) | Для диаспоры за рубежом |

## 6. Конкуренты

Краткий обзор по результатам исследования. Ни один не решает задачу точно под РБ-рынок.

| Инструмент | Суть | Сильная сторона | Слабая сторона |
|-----------|------|----------------|----------------|
| **Marquiz** (marquiz.ru) | Конструктор квизов для СНГ. Кейс: 113 ипотечных заявок за 28 дней | Специфичен для лидогенерации; amoCRM, Bitrix24; AI-генерация | Бесплатный план — 10 лидов; дорожает с объёмом |
| **Madtest.ru** | Рус. конструктор; встраивается в Telegram-бота и VK | Telegram-бот; 16 типов вопросов; VK Pixel | Мало шаблонов для недвижимости; нет AI |
| **Typeform** | Международный конструктор форм | 300+ интеграций; полированный UX | $25/мес за базу; 10 ответов на бесплатном — бесполезно |
| **Involve.me** | No-code воронки; шаблон Property Finder Quiz | Калькулятор стоимости; GDPR; EU-серверы | Нет RU-поддержки; нет интеграций с amoCRM |
| **LeadQuizzes** | Вирусные квизы-личности для соцсетей. Кейс RE/MAX: 1 010 лидов за 2 нед. | Органическое распространение в Facebook-группах | Нет бесплатного плана; нет RU-рынка |

**Вывод:** Рынок занят универсальными конструкторами. Кастомный квиз под РБ-рынок с нативной Telegram-интеграцией — реальная ниша без прямого конкурента.

## 7. Технологии

Выбор основан на исследовании стека: приоритет — нулевая стоимость запуска, простота поддержки.

| Слой | Решение | Обоснование |
|------|---------|-------------|
| **UI квиза** | Vanilla JS + HTML/CSS | Мгновенная загрузка; нет зависимостей; полный контроль дизайна |
| **Хостинг** | Netlify (бесплатный тариф) | Serverless-функции, HTTPS, preview-деплои, 100 ГБ трафика |
| **Бэкенд / уведомления** | Netlify Function → Telegram Bot API | Serverless; бесплатно; Telegram — основной мессенджер риелтора |
| **Хранение заявок** | Google Sheets + Apps Script | Бесплатно; риелтор работает в знакомом интерфейсе |
| **Аналитика** | Yandex Metrica | Бесплатные тепловые карты и запись сессий; популярна в РБ |
| **Резервный канал** | Resend (email, бесплатный тариф) | 3 000 писем/мес; дублирует уведомление если Telegram недоступен |

**Итоговая стоимость инфраструктуры: $0/мес**

## 8. Чего НЕ будет на старте

| Функция | Почему не сейчас |
|---------|-----------------|
| Личный кабинет клиента | Избыточно для MVP; усложняет и стоимость, и поддержку |
| CRM-интеграция | Telegram + Google Sheets закрывают задачу при объёме до 50 заявок/мес |
| Каталог объектов / фильтры | Это маркетплейс, а не инструмент захвата лида |
| Калькулятор ипотеки / оценки | Высокая ценность, но — второй этап после запуска |
| A/B-тестирование | Нужен трафик; добавить через 1–2 месяца после запуска |
| Автоответчик / чат-бот | Страница «Спасибо» достаточна для старта |
| Платёжные функции | Квиз не продаёт, квиз собирает контакты |
| Мультиязычность | Аудитория русскоязычная; EN — второй этап |
| Административная панель | Управление через Netlify + Google Sheets |

## 9. Первые шаги — с чего начать разработку

Последовательность основана на UX-исследовании: сначала доводим мобильную версию, только потом добавляем функции.

### Шаг 1 — Финализировать вопросы квиза (до начала кода)
- Утвердить точные формулировки 7–8 вопросов
- Убедиться, что ни один вопрос не задаёт личное до установления доверия
- Проверить: есть ли опция «пропустить» на необязательных вопросах

### Шаг 2 — Настроить инфраструктуру
- Создать Telegram-бота через BotFather, получить токен и chat_id риелтора
- Создать репозиторий на GitHub, подключить к Netlify
- Подготовить Google Sheet для записи заявок

### Шаг 3 — Сделать HTML/CSS/JS квиза
- Mobile-first: сначала мобильный, потом десктоп
- Один вопрос на экране, крупные кнопки (минимум 44px)
- Прогресс-бар обязателен с первого шага
- Лид-гейт — последним, не первым

### Шаг 4 — Написать Netlify Function
- Принимает данные формы
- Отправляет структурированное сообщение в Telegram
- Дублирует на email через Resend (резервный канал)
- Записывает строку в Google Sheet через Apps Script

### Шаг 5 — Тестирование перед запуском
- Пройти квиз полностью на реальном смартфоне (не в DevTools)
- Проверить получение уведомления в Telegram
- Убедиться, что форма не отправляется без телефона
- Проверить скорость загрузки (цель: <3 сек на мобильном)

### Шаг 6 — Запуск и первые данные
- Подключить Yandex Metrica до запуска
- Настроить отслеживание событий по каждому шагу квиза
- После первых 50 заявок — анализ воронки и первая оптимизация

## 10. KPI и критерии успеха

*Что именно считать, чтобы понять — работает или нет.*

| Метрика | Целевое значение | Откуда данные |
|---------|-----------------|---------------|
| Процент завершения квиза | > 40% (норма в отрасли 40–55%) | Yandex Metrica, события по шагам |
| Конверсия в контакт (лид-гейт) | > 30% от начавших | Yandex Metrica |
| Время прохождения | 2–4 минуты | Metrica / Время сессии |
| Доля мобильного трафика | отслеживать, цель 50%+ | Metrica |
| Заявок в месяц | 20+ (после первой рекламной кампании) | Telegram + Google Sheets |
| Шаг с максимальным отвалом | Выявить и исправить | Пошаговая воронка в Metrica |

## 11. Юридические аспекты

*Часто упускают при первом запуске.*

- **Согласие на обработку персональных данных** — обязательно по законодательству РБ (Закон «О защите персональных данных» №99-З). Чекбокс на форме: «Соглашаюсь на обработку персональных данных» со ссылкой на политику конфиденциальности
- **Политика конфиденциальности** — минимальный документ, описывающий, кто собирает данные, для чего и как долго хранит
- **Telegram как канал хранения** — данные уходят в Telegram; в политике конфиденциальности нужно это упомянуть
- **Email-маркетинг** — если планируется рассылка, нужно явное согласие на получение писем отдельным чекбоксом

## 12. Риски

| Риск | Вероятность | Что делать |
|------|------------|------------|
| Квиз не получает трафик | Высокая без рекламы | Обсудить с риелтором план продвижения до запуска |
| Высокий отвал на конкретном вопросе | Средняя | Yandex Metrica покажет; исправить формулировку |
| Telegram-бот перестаёт работать | Низкая | Resend как резервный email-канал |
| Риелтор не перезванивает быстро | Средняя | Страница «Спасибо» с ожиданием по времени; скорость ответа — зона ответственности клиента |
| Спам-заявки с фейковыми номерами | Средняя | Базовая валидация формата телефона (+375...); капча при необходимости |

## Что могло быть упущено

При работе над проектом стоит отдельно проработать:

1. **План продвижения** — квиз без трафика не работает. Откуда придут первые пользователи: Instagram риелтора? Таргет? SEO? Это должно быть согласовано с клиентом до запуска.
2. **Скрипт первого звонка** — квиз даёт данные, но риелтор должен уметь ими пользоваться. Стоит подготовить короткий чек-лист «что сказать при первом звонке после квиза».
3. **Что происходит с холодными лидами** — те, кто ответил «присматриваюсь». Email-рассылка? Ретаргетинг? Нужна стратегия или эти контакты просто теряются.
4. **Дизайн-система** — цвета, шрифты, стиль. Квиз должен совпадать с брендом риелтора, иначе пользователь подумает, что попал не туда.
5. **Финансовая модель для риелтора** — сколько заявок в месяц нужно квизу, чтобы риелтор считал его окупившимся? Это помогает поставить KPI и оценить успех.
