**Дата:** 2026-07-12
**Проект:** `c:/projects/content-agent`
**Статус:** approved

---

## Что это

Автономный агент для производства и публикации контента в Telegram-канале.
Генерирует текстовые посты на основе аудиторного исследования (`audience.md`), отправляет каждый материал Свете на одобрение через отдельного Telegram-бота, публикует после подтверждения.

Instagram и видео (HeyGen) — следующие этапы, не входят в текущий скоуп.

---

## Требования

### Обязательно (MVP)

- Раз в месяц генерирует план из 20–30 тем (на основе `audience.md`)
- Отправляет план Свете в бот одобрения — она выбирает нужные темы
- По расписанию (или по команде) берёт одну тему → генерирует пост → отправляет на одобрение
- Бот одобрения: кнопки ✅ Опубликовать / ✏️ На правку / ❌ Пропустить
- При ✏️: два пути — инструкция (агент перегенерирует) или готовый текст от Светы
- Публикует одобренный пост в Telegram-канал
- Срочная задача: Света пишет боту «сделай пост про X» → немедленная генерация и одобрение
- К каждому посту генерируется обложка через DALL-E 3 (OpenAI API)
- В боте одобрения черновик показывается вместе с изображением
- Все статусы хранятся в SQLite (не теряются при перезапуске)

### Не входит в MVP

- Instagram (API подключаем отдельным этапом)
- Видео / HeyGen
- Аналитика публикаций

## Архитектура

Детерминированный пайплайн: модули с чёткой ответственностью, бот-оркестратор их связывает.

### Модули

| Файл | Ответственность |
|---|---|
| `bot.py` | Точка входа. Telegram-бот одобрения, оркестратор всего пайплайна |
| `planner.py` | Генерирует месячный план тем (читает `audience.md`, вызывает Claude) |
| `generator.py` | Генерирует один пост по теме (вызывает Claude) |
| `publisher.py` | Публикует готовый пост в Telegram-канал |
| `scheduler.py` | APScheduler: триггер месячного плана + расписание публикаций |
| `storage.py` | SQLite через aiosqlite: план и посты с их статусами |

## Поток данных

### 1. Месячный план

```
scheduler (1-го числа) или команда «/plan»
  → planner.py читает audience.md
  → Claude API: возвращает 20-30 тем с коротким описанием каждой
  → storage.py сохраняет темы со статусом draft
  → bot.py отправляет Свете нумерованный список тем (текстом)
  → Света отвечает текстом: «1 3 5 7» (номера через пробел) или «все»
  → выбранные темы → статус queued
```

### 2. Генерация и одобрение поста

```
scheduler (по расписанию) или команда «/next» или «/post <тема>»
  → generator.py: тема + контекст из audience.md → Claude API
  → возвращает: заголовок, текст поста, хэштеги
  → generator.py: формирует промпт для обложки → DALL-E 3 → скачивает изображение локально
  → storage.py сохраняет черновик + image_path, статус generated
  → bot.py отправляет Свете изображение + текст поста + кнопки:
      [✅ Опубликовать]  [✏️ На правку]  [❌ Пропустить]
```

### 3. Ветки после одобрения

```
✅ Опубликовать
  → publisher.py публикует в канал
  → storage.py: статус published

✏️ На правку
  → bot.py: «Напиши инструкцию или отправь готовый текст»
  → Если инструкция → generator.py перегенерирует → новый черновик (пункт 2)
  → Если готовый текст → сразу publisher.py

❌ Пропустить
  → storage.py: статус skipped
  → scheduler берёт следующую тему
```

### 4. Срочная задача

```
Света пишет: «/post сделай пост про боль с no-show»
  → generator.py: генерирует немедленно
  → то же одобрение (пункт 2)
  → статус не привязан к плану (plan_id = null)
```

## Хранилище (SQLite)

### Таблица `plan`

| Поле | Тип | Описание |
|---|---|---|
| id | INTEGER PK | |
| topic | TEXT | Тема поста |
| description | TEXT | Короткое описание (от planner) |
| month | TEXT | YYYY-MM |
| status | TEXT | draft / queued / generated / published / skipped |
| created_at | DATETIME | |

### Таблица `posts`

| Поле | Тип | Описание |
|---|---|---|
| id | INTEGER PK | |
| plan_id | INTEGER FK | NULL для срочных задач |
| title | TEXT | Заголовок |
| text | TEXT | Текст поста |
| hashtags | TEXT | Строка хэштегов |
| image_url | TEXT | URL изображения от DALL-E 3 (действителен 60 мин) |
| image_path | TEXT | Локальный путь после скачивания (постоянное хранение) |
| status | TEXT | generated / approved / published / skipped |
| created_at | DATETIME | |
| published_at | DATETIME | NULL до публикации |

## Конфигурация (.env)

```
BOT_TOKEN=           # токен бота одобрения (отдельный от Jarvis)
ANTHROPIC_API_KEY=   # ключ Claude API (генерация текста)
OPENAI_API_KEY=      # ключ OpenAI (DALL-E 3, генерация обложек)
CHANNEL_ID=          # @username или -100xxxxxxxxxx Telegram-канала
ADMIN_CHAT_ID=       # chat_id Светы — бот отвечает ТОЛЬКО этому пользователю
POST_TIME=09:00      # время ежедневной публикации (по умолчанию)
TIMEZONE=Europe/Minsk
```

## Команды бота

| Команда | Действие |
|---|---|
| `/plan` | Запустить генерацию месячного плана вручную |
| `/next` | Взять следующую queued тему и сгенерировать пост |
| `/post <текст>` | Срочная задача: сгенерировать пост по произвольной теме |
| `/status` | Показать статистику: сколько тем в плане, опубликовано, в очереди |

## Промпты (зоны ответственности)

**planner.py** передаёт Claude:
- Содержимое `audience.md` (боли, вопросы, пробелы конкурентов)
- Месяц и год
- Инструкцию: «Предложи 20–30 тем для постов в Telegram. Каждая тема — конкретная боль или вопрос из исследования. Формат: номер, тема, одна строка описания.»

**generator.py** передаёт Claude:
- Тему поста
- Краткий контекст аудитории из `audience.md`
- Инструкцию: «Напиши Telegram-пост от первого лица (голос практика). Длина 150–300 слов. Структура: цепляющий первый абзац, суть, конкретный пример или цифра, призыв к действию. В конце — 3–5 хэштегов.»

## Обработка ошибок

- Claude API недоступен → бот сообщает Свете «Ошибка генерации, попробуй /next позже»
- Telegram-канал недоступен при публикации → статус остаётся approved, повтор по команде `/next`
- Бот перезапустился → все статусы в SQLite сохранены, продолжает с того же места

## Расписание по умолчанию

- Публикации: ежедневно по расписанию (конкретное время настраивается через `.env` или команду)
