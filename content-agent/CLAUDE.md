# content-agent (Agent Produsser) — правила работы

Проект: **Agent Produsser** — автономный агент для производства и публикации контента
в Telegram-канале **@missisdev**. Бот одобрения: **@my_prodjuser_bot**.
GitHub: https://github.com/lana1010sveta-dotcom/agent-produsser

Света может называть проект просто **«Продюсер» / «Продюссер»** — это он и есть.

---

## В начале сессии (обязательно)

1. Прочитай `MEMORY.md` — долгосрочные факты о проекте (стек, инфраструктура, ключевые решения).
2. Прочитай `memory/` — дневники за последние 2-3 дня (последний прогресс и контекст).
3. Прочитай `knowledge/architecture.md` — модули, поток данных, команды бота.
4. Если есть `.superpowers/sdd/progress.md` — там ledger задач текущего SDD-плана (что сделано, что нет).
5. `journal.md` — сводный дневник разработки (крупные вехи), читай при появлении нового вопроса про историю проекта.

Без этого шага легко предложить то, что уже сделано, или не заметить открытый баг/TODO.

## В конце сессии (обязательно)

1. Допиши `memory/ГГГГ-ММ-ДД.md` за сегодня: что сделано, какие решения, что осталось.
2. Если новый постоянный факт (инфраструктура, ключевое решение, смена стека) — добавь в `MEMORY.md`, не переписывая файл целиком.
3. Если работаешь по SDD-плану — обнови `.superpowers/sdd/progress.md`.

## Стек

Python 3.14 · python-telegram-bot 22.8 · aiosqlite · APScheduler.
Текст — OpenRouter (`anthropic/claude-sonnet-4-5`), т.к. прямой Anthropic API недоступен
(белорусская карта). Картинки — OpenAI DALL-E 3 (отключены до пополнения баланса,
graceful fallback: пост уходит без картинки). База — SQLite (`data.db`).

## Git

- Коммитить локально можно свободно после каждого рабочего шага.
- **НЕ пушить на GitHub и не деплоить без явной просьбы Светы** — она сама решает,
  когда выкатывать.
- Тесты (`pytest`) должны проходить перед тем, как считать задачу законченной.

## Стиль работы

- Русский язык.
- Конкретика: цифры, факты, имена файлов — без общих фраз.
- Не замалчивать недоделанное — прямо говорить, что не работает / не протестировано.
- Не знаешь точно — спроси, а не придумывай.
- Не усложнять: простое рабочее решение лучше громоздкого.
