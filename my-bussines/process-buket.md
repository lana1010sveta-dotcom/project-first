# Процесс: Обработка заявки на букет — от сайта до доставки

---

## 1. ЧТО ЗАПУСКАЕТ ПРОЦЕСС? (Триггер)

**Триггер:** Клиент заполняет и отправляет форму заявки на сайте.

**Данные, которые приходят с формы:**
- Имя клиента
- Телефон / Telegram
- Адрес доставки
- Желаемая дата и время доставки
- Повод (день рождения, свадьба, просто так…)
- Бюджет или конкретный букет
- Дополнительные пожелания (цвет, цветы, упаковка)

**Где живёт триггер:** Форма на сайте → webhook / email-уведомление → сейчас это читает Светлана вручную.

---

## 2. ПОШАГОВЫЕ ДЕЙСТВИЯ

### Шаг 1 — Получение заявки
- **Кто:** Светлана
- **Что:** Видит уведомление о новой заявке (письмо / звук / случайно замечает)
- **Данные:** Вся форма с сайта
- **Время:** 0 мин (пассивно) + 2–5 мин на прочтение и осмысление
- **Проблема:** Заявка может пролежать незамеченной часами

---

### Шаг 2 — Ответ клиенту в Telegram
- **Кто:** Светлана
- **Что:** Пишет вручную приветственное сообщение: «Ваша заявка принята, мы свяжемся с вами...»
- **Данные:** Имя клиента, детали заказа
- **Время:** 5–10 мин
- **Проблема:** Забыла написать / написала не тот шаблон / ошиблась в имени

---

### Шаг 3 — Копирование данных в Google Sheets
- **Кто:** Светлана
- **Что:** Открывает таблицу, вручную вбивает: имя, телефон, адрес, дату, бюджет, пожелания
- **Данные:** Все поля из формы заявки
- **Время:** 5–10 мин
- **Проблема:** Опечатки при копировании, забытые поля, не та строка

---

### Шаг 4 — Постановка задачи себе на звонок флористу
- **Кто:** Светлана
- **Что:** Ставит напоминание / задачу в заметках / голове «позвонить флористу»
- **Данные:** Детали заказа, бюджет, пожелания
- **Время:** 2–3 мин
- **Проблема:** Задача теряется, напоминание не сработало вовремя

---

### Шаг 5 — Звонок / сообщение флористу
- **Кто:** Светлана
- **Что:** Объясняет флористу: что за букет, бюджет, к какому времени нужна сборка
- **Данные:** Повод, бюджет, цвета, желаемые цветы, дедлайн
- **Время:** 10–20 мин (звонок + ожидание ответа)
- **Проблема:** Флорист не взял трубку, переспрашивает, что-то не так понял

---

### Шаг 6 — Постановка задачи курьеру
- **Кто:** Светлана
- **Что:** Пишет / звонит курьеру: адрес, время, контакт клиента
- **Данные:** Адрес доставки, телефон клиента, окно доставки
- **Время:** 5–10 мин
- **Проблема:** Курьер занят, путает адреса, не видит сообщение

---

### Шаг 7 — Получение фото букета от флориста
- **Кто:** Флорист → Светлана
- **Что:** Флорист присылает фото готового букета
- **Данные:** Фото
- **Время:** Пассивное ожидание + 2–3 мин на пересылку клиенту
- **Проблема:** Флорист забыл прислать, Светлана не увидела / не переслала вовремя

---

### Шаг 8 — Отправка фото клиенту + уточнение времени доставки
- **Кто:** Светлана
- **Что:** Пересылает фото клиенту, спрашивает «всё ок?», уточняет точное время
- **Данные:** Фото букета, окно доставки
- **Время:** 5–10 мин
- **Проблема:** Клиент просит изменения — начинается пересогласование

---

### Шаг 9 — Изменения (если есть)
- **Кто:** Светлана
- **Что:** Доносит правки флористу (переделать), корректирует время у курьера
- **Данные:** Изменённые детали заказа
- **Время:** 10–30 мин (может быть несколько итераций)
- **Проблема:** Самый хаотичный этап — всё вручную, легко что-то упустить

---

### Шаг 10 — Подтверждение доставки
- **Кто:** Курьер → Светлана → Клиент
- **Что:** Курьер сообщает «доставил», Светлана пишет клиенту «ваш букет доставлен»
- **Данные:** Статус доставки, фото (опционально)
- **Время:** 3–5 мин
- **Проблема:** Светлана не получила сигнал от курьера, клиент ждёт подтверждения

---

## 3. ЧТО НА ВЫХОДЕ?

По завершении процесса должно быть:

| Результат | Где фиксируется |
|-----------|-----------------|
| Клиент получил подтверждение заявки | Telegram клиента |
| Клиент одобрил фото букета | Telegram клиента |
| Клиент получил уведомление о доставке | Telegram клиента |
| Заказ записан в таблицу со статусом «Доставлен» | Google Sheets |
| Флорист выполнил ТЗ в срок | Внутренний чат |
| Курьер доставил в нужное время и место | Внутренний чат |
| Отзыв запрошен у клиента | Telegram клиента |

---

## 4. ГДЕ МОЖНО СЛОМАТЬСЯ?

| Точка риска | Что идёт не так | Последствие |
|-------------|-----------------|-------------|
| Заявка пришла ночью | Светлана не видит сразу | Клиент ждёт, уходит к конкуренту |
| Ручной ввод в таблицу | Опечатка в адресе или телефоне | Курьер едет не туда |
| Информация флористу устно | Флорист что-то не так понял | Не тот букет, переделка |
| Несколько заказов одновременно | Светлана путает клиентов | Отправила не то фото не тому |
| Клиент просит правки | Нужно донести всем снова | Хаос, потеря данных, стресс |
| Курьер не отчитался | Светлана не знает статус | Клиент остался без подтверждения |
| Нет шаблона ответа | Каждый раз пишет по-разному | Непрофессиональный вид |

## 5. ПРЕДЛОЖЕНИЕ ПО АВТОМАТИЗАЦИИ

### Архитектура решения

```
[Форма на сайте]
       │
       ▼ webhook (HTTP POST)
[Telegram Bot — центр управления]
       │
       ├──▶ Клиенту: авто-ответ «Заявка принята»
       │
       ├──▶ Google Sheets: авто-запись строки заказа
       │
       ├──▶ Светлане: карточка заказа в Telegram с кнопками
       │          [✅ Принять] [❌ Отклонить]
       │
       ├──▶ Флористу: ТЗ-сообщение с деталями заказа
       │          [📸 Букет готов] — кнопка
       │
       └──▶ Курьеру: маршрутный лист
                  [✅ Доставлено] — кнопка
```

---

### Что автоматизирует каждый элемент

#### Telegram Bot (Python + python-telegram-bot)
| Действие сейчас | Что делает бот |
|-----------------|----------------|
| Светлана пишет клиенту вручную | Бот шлёт шаблон сразу при поступлении заявки |
| Светлана копирует данные в Sheets | Бот пишет строку через Google Sheets API |
| Светлана пишет ТЗ флористу | Бот шлёт карточку заказа в чат флориста |
| Светлана пишет маршрут курьеру | Бот шлёт маршрутный лист в чат курьера |
| Светлана пересылает фото клиенту | Флорист нажал «Готово» → бот переслал фото клиенту |
| Светлана пишет «доставлено» | Курьер нажал «Доставлено» → бот уведомил клиента |
| Светлана просит отзыв | Бот шлёт запрос отзыва через 1 час после доставки |

#### Google Sheets (через Sheets API)
- Автозапись нового заказа при получении заявки
- Обновление статуса при каждом событии (принято → в работе → доставлено)
- История всех заказов без ручного ввода

#### Webhook с сайта
- Форма на сайте отправляет POST-запрос на сервер бота
- Бот получает данные и запускает весь цепочку автоматически

---

### Что останется делать Светлане вручную
- Нажать «Принять заказ» (или отклонить) — 1 кнопка
- Обработать нестандартные ситуации (клиент просит что-то особенное)
- Контроль качества и общение в сложных случаях

**Итог по времени:** с 45–95 мин → до 5–10 мин на заказ

---

### Стек для реализации

| Компонент | Инструмент |
|-----------|-----------|
| Бот | Python + python-telegram-bot |
| Сервер / webhook | Railway или Render (бесплатно) |
| База данных заказов | Google Sheets API |
| Приём заявок с сайта | Webhook endpoint (FastAPI или Flask) |
| Хранение состояний | SQLite или Google Sheets как БД |
| Уведомления | Telegram Bot API |

---

### Следующий шаг

Если принято решение автоматизировать — следующий шаг:
1. Настроить webhook на сайте (или симулировать заявку вручную для теста)
2. Создать Telegram-бота через BotFather
3. Написать обработчик заявки → автоответ клиенту + запись в Sheets
4. Добавить уведомления флористу и курьеру с кнопками статуса
