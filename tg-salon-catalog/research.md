# Исследование ниши: каталог услуг парикмахерской как Telegram Mini App

> Дата: 24.05.2026 · Три агента: UX-аналитик, Продуктовый аналитик, Конкурентная разведка

---

## 1. Анализ конкурентов: 5 реальных TMA для салонов

### 1.1 MastApp (mastapp.ru)

**Что делает:** TMA для онлайн-записи к частным мастерам красоты (парикмахеры, маникюр, массаж). Более 2500 мастеров. Работает полностью внутри Telegram.

**Сильные стороны**
- Нулевое трение: авторизация через Telegram, ничего скачивать
- Гибкие настройки расписания, автонапоминания
- Поддержка предоплаты прямо в мессенджере
- Одна ссылка работает везде: Instagram bio, WhatsApp, VK, QR

**Слабые стороны**
- Один мастер — масштаб на команду ограничен или платный
- Нет публичного каталога: клиент должен уже знать про мастера
- Нет галереи работ внутри TMA
- Бедная аналитика для владельца

**Что мы можем сделать лучше**
- Портфолио мастеров прямо в потоке записи
- Публичный каталог-витрина для поиска нового мастера
- Рейтинг и отзывы перед выбором слота
- Командная работа без доплаты

---

### 1.2 MyAppoint — бесплатная альтернатива YClients в Telegram

**Что делает:** Бесплатный сервис записи для мастеров и небольших салонов через Telegram-бота и Mini App. Управление через естественный язык: «Покажи записи на завтра», «Перенеси Аню на 15:00».

**Сильные стороны**
- Полностью бесплатно — нет тарифов и скрытых платежей
- AI-управление расписанием голосом — революционно удобно для мастера
- Защита от двойного бронирования на уровне архитектуры
- 4 канала: Telegram-бот, веб-страница, виджет для сайта, QR
- Автонапоминания в Telegram

**Слабые стороны**
- Молодой продукт, возможны баги и потеря данных
- Нет онлайн-оплаты и предоплаты
- UI без дизайн-системы — сделан на скорую руку
- Нет CRM: история, заметки, лояльность — ничего
- Масштабируемость не проверена

**Что мы можем сделать лучше**
- Полноценный CRM: история визитов, любимые услуги, день рождения
- Онлайн-оплата и депозит при бронировании (снижает no-show)
- Визуальный onboarding для мастеров без технических навыков
- White-label: цвета и лого конкретного салона

---

### 1.3 Telegram-бот с интеграцией YClients для барбершопов

**Что делает:** Бот с интеграцией в YClients (крупнейшая CRM для бьюти в России). Показывает филиалы, услуги, мастеров, свободные слоты и создаёт запись в YClients. Шаблонное решение для сетей.

**Сильные стороны**
- Расписание синхронизируется с YClients в реальном времени
- Подходит для сетей с несколькими филиалами
- Клиент остаётся в Telegram, вся работа мастера остаётся в YClients

**Слабые стороны**
- Деградирует до «бота с кнопками» — не полноценный Mini App
- Зависимость от YClients: падение = полная остановка
- Длинные цепочки: филиал → услуга → мастер → дата → время
- Нет визуальных карточек мастеров — только имена и кнопки
- Нет онлайн-оплаты без отдельной настройки

**Что мы можем сделать лучше**
- Запомнить любимого мастера, предлагать его следующим разом
- Карточки мастеров с фото в потоке выбора
- «Моя история» — прошлые записи, что стригли, сколько заплатил
- Проактивный оффер: «Ваш мастер освободился раньше»

### 1.4 Beauty-Bot (bot-beauty.ru)

**Что делает:** SaaS-платформа через Telegram и WhatsApp: автозапись, рассылки акций, напоминания о визитах. Позиционируется как «умная запись» с кабинетом администратора.

**Сильные стороны**
- Поддержка Telegram + WhatsApp — охват всей аудитории
- Автонапоминания снижают no-show
- Массовые рассылки акций по базе клиентов
- Настройка под конкретный салон, онбординг за 2–5 дней

**Слабые стороны**
- Инструмент для маркетинга + CRM, а не полноценный Mini App с UI
- Интерфейс клиента — бот с кнопками, нет визуальной привлекательности
- Непрозрачное ценообразование: нужно оставлять заявку
- Нет публичного каталога — только для существующей базы
- Одинаковый интерфейс у всех клиентов

**Что мы можем сделать лучше**
- Визуальный экран выбора мастера с фото и отзывами
- Модуль лояльности с накоплением баллов в TMA
- Прозрачное ценообразование и freemium для одиночных мастеров
- «Живое» расписание без ожидания ответа администратора

### 1.5 SaaS-шаблон TMA «Аналог DIKIDI/YClients» (Telderi)

**Что делает:** Готовый исходный код SaaS-продукта для онлайн-записи: полноценный Mini App (React + Node.js + PostgreSQL) с клиентским экраном и административной панелью. Включает Waitlist.

**Сильные стороны**
- Полноценный Mini App с графическим интерфейсом, не бот с кнопками
- Лист ожидания — редкая фича, которой нет у большинства конкурентов
- Модульный код — можно дорабатывать под конкретный бизнес
- Аналитика, финансы, база клиентов из коробки
- Одноразовая оплата за код, нет вечной подписки

**Слабые стороны**
- Шаблон без поддержки: сами не настроят без разработчика
- «Голый» интерфейс без визуальной идентичности
- Нет встроенной онлайн-оплаты
- Нет мобильного приложения для мастера — только веб-панель

**Что мы можем сделать лучше**
- No-code конструктор для настройки без разработчика
- Галерея работ мастеров в карточке выбора специалиста
- PWA-приложение для мастера с push-уведомлениями
- Onboarding-визард: 5 шагов — и салон готов к записи

## 2. Архитектура экранов и обязательные элементы

### 2.1 Обязательные экраны

**1. Splash / Приветствие**
Показывается при первом открытии (1–2 сек). Логотип, tagline, анимация. Персонализированное «Добро пожаловать, Светлана!» из `initData`. Автопереход или кнопка «Начать».

**2. Главная / Home**
- Приветствие с именем из Telegram
- Карточка ближайшей записи с кнопкой «Отменить»
- Крупный CTA «Записаться»
- Быстрый доступ: Услуги, Мастера, Мои записи

**3. Каталог услуг**
Категории в виде сетки или списка с иконками. Горизонтальные чипы/табы. Каждая услуга: название, описание, цена, длительность.

**4. Карточка услуги**
Фото, название, полное описание, цена, длительность. Список мастеров, выполняющих услугу. MainButton: «Выбрать мастера».

**5. Каталог мастеров**
Фото, имя, специализация, рейтинг. Фильтр по специализации через чипы. Кнопка «Записаться к этому мастеру».

**6. Профиль мастера**
Большое фото, имя, стаж, специализации. Список услуг. Галерея работ (горизонтальный скролл). Отзывы. MainButton: «Записаться».

**7. Выбор даты и времени** *(самый критичный)*
- Горизонтальный скролл по датам или месячный календарь
- Сетка слотов: свободный / занятый / выбранный
- Стики-шапка с выбранным мастером и услугой
- MainButton «Продолжить» активируется только после выбора

**8. Подтверждение записи**
Услуга + мастер + дата/время + стоимость. Адрес с кнопкой «Открыть в картах». Поле для комментария. MainButton: «Подтвердить запись».

**9. Экран успеха**
Иконка + поздравление. Краткое резюме. Кнопка «Добавить в календарь». Кнопка «На главную». Без MainButton — два равнозначных действия.

**10. Мои записи**
Вкладки «Предстоящие» / «История». Карточки: услуга, мастер, дата, статус-бейдж. Кнопки: «Отменить», «Записаться снова». Empty state с CTA.

**11. Отмена записи**
Информация об отменяемой записи. Предупреждение о политике отмены. «Отменить запись» (красный) + «Не отменять». После отмены — toast + уведомление в чат.

**12. Профиль пользователя** *(желательно)*
Имя и аватар из Telegram. История записей. Настройки уведомлений. Программа лояльности / бонусы.

---

### 2.2 Обязательные UI-элементы

**Навигация**
- Bottom Navigation Bar (4–5 пунктов: Главная, Услуги, Мастера, Записи, Профиль)
- `BackButton` SDK — обязателен на каждом вложенном экране, синхронизирован с router
- `MainButton` SDK — для главного CTA. Один экран = одно действие

**Карточки**
- Услуга: изображение, название, цена, длительность, кнопка
- Мастер: фото, имя, специализация, рейтинг
- Запись: дата/время, услуга, мастер, статус-бейдж, кнопки действий

**Кнопки**
- Минимальный размер: 44×44px
- Активные глаголы: «Записаться», «Выбрать», «Подтвердить», «Отменить запись»
- Состояния: default / pressed / disabled / loading
- Деструктивные действия — красный цвет + подтверждение

**Состояния загрузки**
- Skeleton-карточки вместо спиннера
- `MainButton.showProgress()` при отправке запроса
- Pull-to-refresh для списков

**Слоты времени**
- Явные визуальные различия: свободный / занятый / выбранный
- Сетка 3–4 слота в строку
- Занятые — некликабельны

**Пустые состояния**
- Каждый список: иллюстрация + текст + CTA
- «У вас пока нет записей → Записаться»

**Toast / Snackbar**
- «Запись отменена», «Запись создана», «Слот уже занят»
- 2–3 секунды, не блокирующие интерфейс

---

### 2.3 Основные пользовательские сценарии

**Flow 1: Запись на услугу** *(основной)*
```
Главная → Каталог услуг → Выбор услуги → Выбор мастера
→ Выбор даты/слота → Подтверждение → Экран успеха
+ уведомление в Telegram-чате бота
```
Максимум 5–6 кликов. Быстрый путь для постоянного клиента: Главная → Любимый мастер → Дата.

**Flow 2: Просмотр мастеров**
```
Главная → Мастера → [фильтр] → Профиль мастера
→ [Записаться к мастеру] → Flow 1 с предустановленным мастером
```

**Flow 3: Отмена записи**
```
Главная (карточка записи) → [Отменить]
ИЛИ Мои записи → Предстоящие → Карточка → [Отменить]
→ Подтверждение отмены → Toast → Уведомление в чат
```

**Flow 4: Повторная запись** *(ключевая retention-механика)*
```
Мои записи → История → [Записаться снова]
→ Мастер и услуга предзаполнены → Выбор даты → Подтвердить
```

**Flow 5: Перенос записи**
```
Мои записи → Предстоящие → [Перенести]
→ Новая дата (мастер и услуга зафиксированы) → Подтвердить
```

### 2.4 Критические ошибки (антипаттерны)

1. **Более 3 шагов до выбора слота** — каждый лишний экран убивает конверсию
2. **Нет контекста при выборе слота** — пользователь забыл, что выбрал. Всегда стики-шапка
3. **Занятые слоты кликабельны** — самый быстрый способ убить доверие
4. **BackButton не синхронизирован** — пользователь нажимает back, TMA закрывается целиком
5. **Нет синхронизации темы** — ярко-белый экран в тёмном Telegram = «сломанное приложение»
6. **Форма с 10+ полями** — имя уже есть из Telegram; телефон — только через `requestContact()`
7. **Timepicker-барабан** — неудобно; нужна кастомная сетка доступных слотов
8. **Нет empty state и error state** — белый экран = UX-баг
9. **Подтверждение только внутри TMA** — бот должен слать сообщение в чат + напоминание
10. **Деструктивные действия без подтверждения** — случайные отмены убивают доверие
11. **Горизонтальные карусели без защиты от системного свайпа** — TMA закрывается случайно
12. **MainButton без loading state** — двойной клик = двойная запись

### 2.5 Технические особенности Telegram Mini App

**Безопасность**
- `initData` — верифицировать на backend через HMAC-SHA-256. Без верификации — подделка запросов
- `initDataUnsafe.user.first_name` — только для UI (приветствие), никогда для авторизации

**Нативные компоненты SDK**
```js
// MainButton
WebApp.MainButton.setText('Подтвердить запись')
WebApp.MainButton.show()
WebApp.MainButton.showProgress() // при запросе — блокирует повторные нажатия
WebApp.MainButton.hideProgress() // после ответа

// BackButton
WebApp.BackButton.show()
WebApp.BackButton.onClick(() => router.back())
WebApp.BackButton.hide() // на корневых экранах

// SecondaryButton (Bot API 7.10+) — второе действие рядом с MainButton
```

**Haptic Feedback**
```js
WebApp.HapticFeedback.selectionChanged()        // выбор слота
WebApp.HapticFeedback.impactOccurred('medium')  // нажатие «Записаться»
WebApp.HapticFeedback.notificationOccurred('success') // успешная запись
WebApp.HapticFeedback.notificationOccurred('error')   // слот занят
```

**Тема**
```js
const theme = WebApp.themeParams
// Применять CSS-переменные: --bg-color, --text-color, --button-color
// Слушать событие themeChanged — пользователь может переключить тему во время работы TMA
```

**Viewport**
```js
WebApp.expand() // при старте — развернуть на весь экран
// iOS: добавлять padding-bottom: env(safe-area-inset-bottom)
// к bottom navigation и MainButton-контейнеру
```

**Запрос контакта**
```js
WebApp.requestContact((shared, contact) => {
  if (shared) savePhoneNumber(contact.phoneNumber) // один тап вместо ввода
})
```

**Уведомления через бота**
- Подтверждение записи — сразу после создания
- Напоминание за 24 часа и за 2 часа
- Уведомление об отмене мастером
- Inline-кнопки: «Подтвердить присутствие», «Перенести», «Открыть запись»

**Производительность**
- FCP < 1.5 сек (Telegram показывает лоадер пока TMA грузится)
- Skeleton screens, lazy load изображений
- Кэшировать мастеров и услуги локально (меняются редко)
- Слоты запрашивать только при выборе конкретного мастера и даты

## 3. Рынок, тренды и возможности

### 3.1 Рынок и контекст

Глобальный рынок ПО для салонов красоты: **$1,01 млрд в 2025 → $1,69 млрд к 2030** (CAGR 10,9%).

Ключевая статистика:
- 82% онлайн-бронирований — с мобильных телефонов
- 60%+ клиентов предпочитают записываться через приложение, а не звонить
- Автонапоминания снижают no-show на 27%
- Салоны теряют до 20% записей из-за плохой организации расписания

**Почему клиенты пишут в Telegram вручную?**
Запись = доверие и личный контакт. Клиент хочет подтверждения от живого человека, уточнить нюансы. Системы бронирования воспринимаются как холодные. TMA решает это — разговор и бронирование в одном месте.

---

### 3.2 Основные конкуренты (не TMA)

**YCLIENTS** — мощная CRM, но дорогая и сложная. Из реальных отзывов: «приложение тормозит», «пуши не приходят», «поддержка молчит», «малому бизнесу переплачивать не за что».

**Fresha** — бесплатная базовая версия, но скрытые комиссии: 20% с каждого нового клиента через платформу. При высоком обороте становится убийственным.

**Booksy** — мощный маркетплейс, SEO-трафик, популярен у барберов. Жалобы: двойное списание, давление на апгрейд, слабая поддержка.

---

### 3.3 Уникальные возможности TMA

| Преимущество | Эффект |
|---|---|
| Нулевое трение при входе (уже авторизован) | Конверсия выше на 20–30% |
| Нет установки приложения | Клиент не захламляет телефон |
| Push бесплатно через Telegram | Open rate 90%+ против 20–30% у email |
| Вирусное распространение: переслать другу в 1 тап | Органический реферальный канал |
| Telegram Payments — оплата без редиректа | Меньше брошенных корзин |
| Имя клиента из профиля автоматически | Персонализация с первого экрана |
| Разработка в 3–5 раз дешевле нативного приложения | Быстрый выход на рынок |
| 7-дневный retention 30–50% | Против 5–10% у веб-сервисов |

---

### 3.4 Тренды 2024–2025

**AI и персонализация**
- AI-подбор стрижки и цвета: анализ формы лица, виртуальная примерка
- AI-расписание: предсказание пиковых часов, умное перераспределение при отменах
- AI-консультант перед записью: «какой эффект хотите?» → рекомендация услуги и мастера

**Программы лояльности**
- Лояльность в beauty выросла до 42% в 2025 (+10% за два года)
- Тренд: геймификация (баллы, уровни, дропы), а не просто скидки

**Автоматизация вейтлиста**
При отмене — мгновенное предложение слота следующему. Возврат 60–70% иначе потерянных часов.

**Conversational Commerce**
Рынок записи через мессенджеры достигнет $290 млрд к 2025 году. Telegram, WhatsApp, Instagram — основные каналы для малого бизнеса в beauty.

---

### 3.5 Незакрытые потребности (голубой океан)

1. **Портфолио конкретного мастера, а не салона** — клиент хочет записаться к Марине, потому что видел её работы. Нет решений с личным портфолио мастера внутри booking-инструмента.

2. **Умный вейтлист с мгновенным предложением** — при отмене автоматическое сообщение в Telegram всем в листе ожидания. Первый подтвердил — слот закрыт.

3. **AI-консультация до записи** — «хочу что-то обновить» — это не услуга. Мини-квиз перед записью повышает конверсию и снижает недовольство результатом.

4. **Проактивные напоминания на основе истории** — «Вы красили волосы 6 недель назад — пора на тонирование?». Никто из TMA-решений этого не делает.

5. **Видео-презентация мастера** — 15–30 сек: как мастер работает, какова атмосфера. Тревожность клиента снижается, доверие растёт.

6. **Реферальная механика «приведи подругу»** — Telegram идеален для шеринга, но никто не использует нативный шеринг полностью.

7. **Решение для самозанятых мастеров** — YCLIENTS и Fresha заточены под юрлицо. Миллионы частных мастеров на дому или в аренде кресла не охвачены.

8. **Сбор отзывов через Telegram** — сообщение через 3 часа после визита: «Как всё прошло? ⭐⭐⭐⭐⭐» — конверсия в отзыв несравнимо выше, чем email или ссылка на Google.

### 3.6 Рекомендации для нашего проекта

**Приоритет 1 — Дифференциаторы (делают нас уникальными)**

- **Портфолио мастера внутри TMA** — галерея работ до/после прямо в карточке
- **Умный вейтлист** — Telegram-сообщение всем при отмене, первый подтвердил — слот закрыт
- **Реферальная механика** — нативный шеринг Telegram: «поделись ботом — получи бонус»

**Приоритет 2 — Конверсионные фичи**

- **Сбор отзывов через Telegram** — авто-сообщение через 3–4 часа после визита
- **Проактивные напоминания** — на основе истории услуг клиента
- **Telegram Payments для депозита** — снижает no-show до 5% и ниже

**Приоритет 3 — Точечные AI-фичи**

- **Мини-квиз перед записью** — 3–4 вопроса → рекомендация услуги и мастера
- **AI-напоминание об уходе** — после услуги: «рекомендуем вернуться через 6 недель»

**Модель монетизации — избегать ошибок конкурентов**

- Не брать % с новых клиентов (главная причина ненависти к Fresha)
- Простой фиксированный тариф: от 490 до 990 руб./мес
- Бесплатный период 30 дней без кредитки
- Freemium для самозанятых: 1 мастер — бесплатно (с брендингом сервиса)

**Целевой сегмент для быстрого старта**

Самозанятые мастера (парикмахеры, бровисты, мастера ногтей) — YCLIENTS им дорог, Fresha берёт комиссию. Недообслуженный рынок с миллионами потенциальных пользователей.

## 8. Экспертная оценка идеи

> **Идея:** Telegram Mini App для парикмахерской — каталог услуг, каталог фото причёсок, большие кнопки для записи и выбора услуг. Открывается на телефоне.

---

### 8.1 Взгляд UX-дизайнера

#### Типичные ошибки новичков

**«Красивый каталог» вместо «быстрой записи»**
Самая частая ловушка — делают красивую витрину с фото, но путь до кнопки «Записаться» занимает 5+ кликов. Пользователь пришёл не смотреть, а записаться. Каталог фото — вторичная цель; первичная — слот в расписании.

**Большие кнопки ≠ удобство**
«Большие кнопки» — правильный инстинкт, но часто реализуется неверно: делают крупные, но расплывчатые («Далее», «Перейти» вместо «Записаться на стрижку»). Кнопка должна быть крупной И содержать конкретный глагол действия.

**Фото причёсок без контекста**
Каталог фото без привязки к мастеру и цене бесполезен. Пользователь видит красивую стрижку, но не понимает: кто делает, сколько стоит, когда можно прийти. Каждое фото должно вести к записи.

**Потеря контекста при переходах**
Пользователь выбирает стрижку → переходит к выбору мастера → забывает что выбрал. Выбранная услуга должна быть видна на каждом следующем шаге (стики-шапка или мини-саммари вверху).

**Нет skeleton и empty state**
Белый экран пока грузятся фото = ощущение сломанного приложения. Skeleton-карточки обязательны. Пустое расписание — не баг, но нужно говорить пользователю что делать дальше.

#### Что обязательно должно быть

- Путь от входа до «Запись подтверждена» — **максимум 4 тапа**
- Каждое фото причёски → кнопка **«Записаться на эту стрижку»** (предзаполняет услугу)
- Профиль мастера с **реальными фото его работ**, не стоковые картинки
- **Ближайший свободный слот** прямо на главной («Ближайшая запись: завтра в 14:00»)
- **Подтверждение в чат-ботом** после записи — не только экран успеха внутри TMA
- **BackButton синхронизирован с роутером** — иначе TMA закрывается при системном «назад»

#### Что бесит пользователей

- Поле «Введите ваше имя» — оно уже есть в Telegram
- Занятые слоты кликабельны, а потом — ошибка
- Карусель фото нечаянно свайпнул — TMA закрылась
- Долгая загрузка без визуальной обратной связи
- Нет кнопки «Записаться снова» — нужно проходить весь путь заново
- Отмена записи одним кликом без подтверждения

---

### 8.2 Взгляд маркетолога

#### Что заставит записаться

**Социальное доказательство на первом экране**
Не логотип и слоган, а: «★ 4.9 · 127 отзывов · Работаем с 2018». Клиент принимает решение в первые 3 секунды. Дай ему причину остаться.

**Реальные фото, а не сток**
Пользователи умеют отличать сток от реальных работ. 20 реальных фото мастера конвертируют лучше, чем 100 красивых стоковых. Аутентичность — главный триггер доверия в beauty.

**«Следующий свободный слот» как CTA**
Вместо абстрактного «Записаться» — конкретика: «Записаться: завтра 11:00». Конкретная дата снижает барьер принятия решения.

**Цена видна до начала записи**
Скрытая цена («уточняйте у мастера») убивает конверсию. Покажи хотя бы диапазон: «Стрижка: от 800 руб.» Клиент должен знать бюджет до выбора слота.

**Фото мастера + имя + специализация**
«Записаться» к безымянному мастеру — холодно. «Записаться к Марине, 8 лет стажа, специалист по окрашиванию» — тепло и доверие. Конкретный человек конвертирует лучше абстрактной «услуги».

**Срочность без манипуляций**
«Осталось 2 свободных слота на этой неделе» — если это правда, показывай. Реальный дефицит работает. Фейковый таймер разрушает доверие навсегда.

#### Элементы, повышающие конверсию

| Элемент | Эффект |
|---|---|
| Предоплата/депозит при записи | No-show снижается с 20–30% до 5% |
| Напоминание за 2 часа в Telegram | Ещё −15% неявок |
| «Записаться снова» в 1 тап | +30–40% повторных записей |
| Кнопка «Поделиться» после успешной записи | Виральность без бюджета |
| Авто-отзыв через 3 часа после визита | Накопление репутации |
| Акция для первого визита | Снижает барьер для новых клиентов |
| Реферальный бонус за приведённого друга | CAC снижается |

### 8.3 Взгляд Telegram-разработчика

#### Ограничения, которые нужно знать заранее

**Производительность**
TMA открывается в WebView внутри Telegram. Бюджет на загрузку — 1.5 сек до First Contentful Paint. Тяжёлые фото-галереи требуют lazy-load + WebP + кэширование, иначе пользователь видит пустой лоадер и уходит.

**Жесты и горизонтальный свайп**
На iOS горизонтальный свайп конфликтует с жестом закрытия TMA. Если делаешь карусель фото причёсок — обязательно обрабатывать `touchstart`/`touchmove` и предотвращать всплытие. Иначе пользователь листает фото и случайно закрывает приложение.

**MainButton — один на экран**
Распространённая ошибка: кастомная большая кнопка внизу И включённый MainButton одновременно. Они конфликтуют. Выбери одно: либо Telegram `MainButton` (нативно, всегда внизу), либо кастомная кнопка (полный контроль над дизайном). Для первого проекта — кастомная проще.

**Тема — обязательная синхронизация**
Без `themeParams` у пользователей с тёмной темой TMA будет выглядеть как белая вспышка. Минимум: CSS-переменные для фона и текста из `WebApp.themeParams`. Для каталога фото особенно важно — тёмный фон под фотографиями выглядит профессиональнее.

**Нет фонового выполнения**
TMA работает только пока открыта. Напоминания — только через бота (`sendMessage`). Таймер внутри TMA бессмысленен — закроется вместе с приложением.

**Safe area**
`WebApp.expand()` при старте — обязательно. Без этого TMA открывается на половину экрана. После expand — полный экран, но нижняя панель Telegram остаётся. Добавить `padding-bottom: env(safe-area-inset-bottom)` к кнопкам внизу.

**Безопасность**
Backend ОБЯЗАН верифицировать `initData` через HMAC-SHA-256. Без этого любой может отправить запрос с чужим `user_id` и записаться от чужого имени.

#### Что работает хорошо

- **Haptic feedback** — `selectionChanged()` при выборе слота, `notificationOccurred('success')` при подтверждении. Делает приложение «живым»
- **`requestContact()`** — один тап для получения телефона вместо ввода вручную
- **BackButton** — `WebApp.BackButton.show()` + синхронизация с router. Пользователи Telegram его ожидают
- **Telegram Payments** — встроенная оплата без редиректа. Работает надёжно
- **`initDataUnsafe.user.first_name`** — имя пользователя для персонализированного приветствия

#### Чего лучше не делать

- `window.open()` — непредсказуемо. Использовать `WebApp.openLink()` или `WebApp.openTelegramLink()`
- iframe внутри TMA — Telegram блокирует вложенные фреймы
- Хранить auth-токен без валидации `initData` на сервере
- Сложные 60fps-анимации — WebView не GPU-оптимизирован как нативное приложение
- Горизонтальный full-screen свайп без обработки конфликтов жестов

### 8.4 Общий список рекомендаций

#### Сделать обязательно (влияет на запуск)

1. Путь записи — максимум 3–4 шага, всё остальное опционально
2. Предзаполнять имя из Telegram, не просить вводить
3. Вызывать `WebApp.expand()` при загрузке
4. Все цвета синхронизировать через `WebApp.themeParams`
5. `BackButton` синхронизировать с роутером на каждом экране
6. Skeleton-карточки вместо спиннера при загрузке фото
7. После записи — бот отправляет сообщение в чат с деталями
8. Занятые слоты — серые и некликабельные

#### Сделать до первых клиентов

9. Каждое фото причёски → кнопка «Записаться на эту стрижку»
10. Профиль мастера с реальными работами (не сток)
11. Ближайший свободный слот — показывать на главной
12. Toast-уведомление после каждого действия (запись, отмена)
13. Empty state для пустого расписания и пустой истории
14. Обработать конфликт горизонтального свайпа в галерее

#### Сделать для роста

15. Автонапоминание за 2 часа до визита через бота
16. «Записаться снова» в 1 тап из истории визитов
17. Авто-отзыв через 3 часа после визита
18. Депозит при записи через Telegram Payments
19. Кнопка «Поделиться» на экране успешной записи
20. Реферальная механика: приведи подругу — получи бонус

#### Чего не делать никогда

- Поле «Введите ваше имя» — имя есть в Telegram
- Стоковые фото причёсок вместо реальных работ мастера
- Занятые слоты кликабельными
- Деструктивные действия без подтверждения
- `window.open()` вместо `WebApp.openLink()`
- Горизонтальная карусель без обработки touch-конфликтов
