# Brief: Telegram Mini App — Каталог парикмахерской (РБ)

> Версия 1.0 · Составлен на основе research.md · 24.05.2026

---

## Контекст

Приложение для парикмахера из Беларуси. Открывается из Telegram на телефоне.  
Клиент — физическое лицо: один мастер или небольшой салон.  
Язык интерфейса: **русский**. Валюта: **BYN**.  
Стиль: нативный Telegram — цвета, шрифты и компоненты из `themeParams`.

---

## Экраны первой версии

### Экран 1 — Главная (Home)

**Что видит пользователь**
- Название салона — крупно; адрес — помельче, под названием
- Tagline: 1 строка («Стрижки, окрашивание, уход»)
- Рейтинг и число отзывов (★ 4.9 · 47 отзывов) — если есть
- Блок «Ближайший свободный слот»: «Завтра, 14:00 → кнопка Записаться»
  - Данные загружаются из API при каждом открытии главной; после создания или отмены записи блок обновляется автоматически — показывает актуальный первый свободный слот
- Три крупные карточки-кнопки:
  - 🗂 Услуги и цены
  - 📸 Фото работ
  - 📅 Мои записи

**Кнопки (min 44px высота)**
- `Записаться` (акцентный цвет, крупная, полная ширина) → Экран 3 (Каталог услуг)
- `Услуги и цены` → Экран 3
- `Фото работ` → Экран 4
- `Мои записи` → Экран 5

**Переходы**
- Все три карточки и кнопка «Записаться» — ведут к соответствующим разделам
- BackButton скрыт (корневой экран)

### Экран 3 — Каталог услуг

**Что видит пользователь**
- Горизонтальные чипы категорий:  
  «Все» · «Стрижка» · «Окрашивание» · «Уход»
- Список карточек услуг (по одной в строку):
  - Иконка или мини-фото
  - Название услуги — крупно
  - Описание — 1 строка серым текстом
  - Цена в BYN (напр. «от 25 руб.»)
  - Продолжительность (напр. «45 мин»)
  - Кнопка `Выбрать` справа

**Кнопки**
- Чипы категорий → фильтрация списка (без перехода)
- `Выбрать` на карточке → Экран 3а (Детали услуги)

**Переходы**
- BackButton → Главная

### Экран 3а — Детали услуги

**Что видит пользователь**
- Фото-пример работы (если есть) или иллюстрация, полная ширина
- Название услуги — крупно
- Описание — 3–5 строк (что входит, результат)
- Цена и длительность — в двух колонках, крупно
- Стики-шапка при скролле: «Услуга: Стрижка · 25 руб.»

**Кнопки**
- `Записаться на эту услугу` (полная ширина, акцентный) → Экран 6 (Выбор даты)

**Переходы**
- BackButton → Экран 3 (Каталог)
- Кнопка → Экран 6 с предзаполненной услугой

### Экран 4 — Фото работ (галерея)

**Что видит пользователь**
- Горизонтальные чипы: «Все» · «Стрижка» · «Окрашивание» · «Укладка» · «Борода»
- Сетка фото 2 колонки (квадратные превью)
- При тапе на фото → lightbox (полноэкранный просмотр)

**В lightbox**
- Фото на весь экран
- Название услуги и цена — снизу
- Кнопка `Записаться на такую же` (снизу, полная ширина) → Экран 6

**Переходы**
- BackButton → Главная
- Lightbox: свайп вверх или крест → закрыть (остаться в галерее)
- Кнопка в lightbox → Экран 6 с предзаполненной услугой

### Экран 5 — Мои записи

**Что видит пользователь**
- Две вкладки: `Предстоящие` · `История`

**Вкладка «Предстоящие»:**
- Карточка каждой записи:
  - Дата и время — крупно
  - Услуга
  - Статус-бейдж: `Подтверждена` / `Ожидает`
  - Кнопка `Отменить` — текстовая, красная
- Пустое состояние: иллюстрация + «Записей пока нет» + кнопка `Записаться`

**Вкладка «История»:**
- Те же карточки, статус `Завершена`
- Кнопка `Записаться снова` — текстовая, акцентная

**Кнопки**
- `Отменить` → Экран 5а (Подтверждение отмены)
- `Записаться снова` → Экран 6 с предзаполненной услугой и мастером
- `Записаться` (empty state) → Экран 3

**Переходы**
- BackButton → Главная

### Экран 5а — Подтверждение отмены

**Что видит пользователь**
- Модальный лист (bottom sheet):
  - «Отменить запись?»
  - Детали: услуга, дата, время

**Кнопки**
- `Да, отменить` (красный, полная ширина)
- `Не отменять` (вторичный, полная ширина)

**Переходы**
- После отмены: toast «Запись отменена» → Экран 5
- «Не отменять» → закрыть sheet, остаться на Экране 5

### Экран 6 — Выбор даты и времени

**Что видит пользователь**
- Стики-шапка: выбранная услуга + цена (не теряется при скролле)
- Горизонтальный скролл дат: 14 дней вперёд
  - Формат: «Пн 26» — активная / «Вт 27» — недоступная (серая)
- Сетка слотов (3 в строку, крупные):
  - Зелёные / акцентные — свободные
  - Серые, некликабельные — занятые
  - Подсвеченный border — выбранный

**Кнопки**
- `Продолжить` (полная ширина, акцентный) — активируется только после выбора слота → Экран 7

**Переходы**
- BackButton → откуда пришли (Детали услуги / Галерея / Мои записи)

### Экран 7 — Подтверждение записи

**Что видит пользователь**
- Заголовок: «Проверьте запись»

  - Услуга
  - Дата и время
  - Цена в BYN

**Кнопки**
- `Подтвердить запись` (полная ширина, акцентный, крупный) → Экран 8

**Переходы**

- `Изменить время` → Экран 6

## Требования к дизайну

### Стиль
- **Цвета**: исключительно через `WebApp.themeParams` — авто-адаптация к светлой/тёмной теме Telegram
- **Акцентный цвет**: `button_color` из темы (стандартный синий Telegram или кастомный)
- **Шрифты**: системный (SF Pro / Roboto — то, что использует устройство)
- **Фон**: `bg_color` из темы
- **Карточки**: `secondary_bg_color` с `border-radius: 12px`

### Размеры и отступы
- Кнопки: **минимум 48px высота** (запас над требованием 44px)
- Горизонтальные отступы контента: **16px**
- Между карточками: **12px**
- Bottom navigation: высота 56px + `safe-area-inset-bottom`

### Изображения
- Формат: **WebP** (размер в 2× меньше JPEG при том же качестве)
- Превью в галерее: 160×160px
- Фото мастера на главной: ширина 100%, высота 180–220px, `object-fit: cover`
- Skeleton при загрузке: серые прямоугольники той же формы что и контент

### Ощущение «приложения, не сайта»
- `WebApp.expand()` при старте — полноэкранный режим
- Нет видимого URL, нет browser chrome
- Переходы: slide (правая панель выезжает) или fade — не reload страницы
- Haptic feedback при выборе слота и подтверждении записи
- MainButton Telegram не используем — своя кнопка в потоке (больше контроля над дизайном)

## Технические решения

### Frontend
- **Стек**: Vanilla JS + HTML/CSS (или React — если нужна сложная навигация)
- **Роутинг**: hash-роутер (`#home`, `#services`, `#gallery`) — SPA без перезагрузок
- **SDK**: `@twa-dev/sdk` или прямое подключение `telegram-web-app.js`
- **Изображения**: lazy loading через `IntersectionObserver`

### Backend (минимальный MVP)
- Статичные данные: услуги, фото, слоты — JSON-файл (для самого первого запуска)
- Бот: python-telegram-bot или Telegraf (Node.js)
- API для записи: `/book` принимает `{user_id, service_id, slot}`, проверяет `initData`
- Хранение записей: SQLite или PostgreSQL

### Уведомления
- Сразу после записи: бот шлёт сообщение в чат
- За 2 часа до визита: cron или celery task

## Чего НЕ будет в первой версии

| Что | Почему нет |
|---|---|
| Онлайн-оплата и депозит | Требует подключения к платёжной системе — отдельный этап |
| Несколько мастеров | Один мастер — MVP; масштабирование после валидации |
| Программа лояльности и баллы | Сложность без доказанного retention |
| AI-квиз / AI-рекомендации | Переусложнение для старта |
| Регистрация и пароль | Авторизация через Telegram — достаточно |
| Отзывы внутри приложения | Сбор отзывов через бота — достаточно для MVP |
| Панель администратора (web) | Управление через бота в Telegram |
| Перенос записи | Сначала отмена + новая запись; reschedule — в v2 |
| Вейтлист | После MVP, когда появится реальный спрос |
| Мультиязычность | Только русский в v1 |

## Порядок разработки

### Этап 1 — Скелет и навигация (день 1–2)
1. Подключить `telegram-web-app.js`, вызвать `expand()`, настроить `themeParams`
2. Сделать bottom navigation с 4 вкладками
3. Реализовать hash-роутер: переходы между экранами без перезагрузки
4. Синхронизировать `BackButton` с роутером

### Этап 2 — Главная и каталог (день 3–4)
5. Главная: фото, имя, tagline, три карточки, кнопка «Записаться»
6. Каталог услуг: чипы + список карточек из JSON
7. Детали услуги: фото, описание, цена, кнопка

### Этап 3 — Галерея (день 5)
8. Сетка фото 2 колонки с lazy load
9. Lightbox с кнопкой «Записаться на такую же»
10. Обработать конфликт горизонтального свайпа

### Этап 4 — Запись (день 6–7)
11. Экран выбора даты: горизонтальный скролл + сетка слотов
12. Стики-шапка с выбранной услугой
13. Экран подтверждения: саммари + кнопка
14. Экран успеха: иконка, резюме, кнопки

### Этап 5 — Мои записи и бот (день 8–9)
15. Экран «Мои записи»: вкладки, карточки, пустые состояния
16. Подтверждение отмены (bottom sheet)
17. Backend: API `/book`, валидация `initData`, сохранение в БД
18. Бот: сообщение после записи, напоминание за 2 часа

### Этап 6 — Полировка (день 10)
19. Skeleton-карточки для всех списков
20. Haptic feedback
21. Toast-уведомления
22. Тест на реальном iPhone и Android в Telegram
23. Тест тёмной и светлой темы

## Метрики успеха MVP

- Пользователь записывается за ≤ 4 тапа
- FCP (First Contentful Paint) < 1.5 сек
- No-show не более 15% (без депозита)
- 50%+ пользователей возвращаются для повторной записи в течение 60 дней
